# Privacy Policy

**Last updated**: April 7, 2026

TabRest is committed to protecting your privacy. This policy explains what data we collect (or don't collect) and how it's used.

## Summary

- ✅ **No personal data collected**
- ✅ **No external servers or network requests**
- ✅ **All data stored locally in your browser**
- ✅ **No tracking, analytics, or telemetry**
- ✅ **Open source and auditable**

## Data Collection

### We Do NOT Collect

TabRest does **not** collect, store, or transmit:

- ❌ Your browsing history
- ❌ Website content or passwords
- ❌ Personal information (name, email, etc.)
- ❌ Usage analytics or telemetry
- ❌ Crash reports
- ❌ Device information
- ❌ Location data
- ❌ Any data whatsoever

### We DO Store Locally

TabRest stores the following data **only on your device**:

| Data Type    | Storage Location       | Purpose                                   | Synced? |
| ------------ | ---------------------- | ----------------------------------------- | ------- |
| Settings     | `chrome.storage.sync`  | User preferences (timer, threshold, etc.) | Yes\*   |
| Whitelist    | `chrome.storage.sync`  | Protected domains                         | Yes\*   |
| Tab activity | `chrome.storage.local` | Last activity time per tab (for LRU)      | No      |
| Statistics   | `chrome.storage.local` | Tabs unloaded count, memory saved         | No      |

\*Synced via Chrome Sync if you've enabled Chrome sync in your browser settings. We do not control or access this sync data.

## Chrome Permissions

TabRest requires these permissions:

### `tabs` - "Read your browsing history"

**What it actually does**:

- Read tab URLs to match against whitelist
- Read tab state (pinned, audio, active)
- Call `chrome.tabs.discard()` to unload tabs

**What it does NOT do**:

- Send your browsing history anywhere
- Track which sites you visit
- Store your browsing history

**Why Chrome shows this label**: The `tabs` permission gives access to tab URLs. Chrome's permission labels are generic and don't reflect actual usage.

### `storage` - Store data locally

**What it does**:

- Save settings (timer, threshold, protections)
- Save whitelist entries
- Store tab activity for LRU tracking
- Track statistics (tabs unloaded, memory saved)

**All data stays on your device.** No external storage or cloud services.

### `alarms` - Schedule periodic tasks

**What it does**:

- Schedule periodic tab checks (every 1 minute)
- Schedule memory threshold checks (every 30 seconds)

**Why needed**: Chrome service workers terminate when idle. Alarms wake the service worker to perform checks.

### `system.memory` - Read memory usage

**What it does**:

- Read total and available system RAM
- Calculate memory usage percentage
- Trigger unloading when threshold exceeded

**What it does NOT do**:

- Access any other system information
- Collect or store memory data
- Send memory data anywhere

### `contextMenus` - Add right-click menu items

**What it does**:

- Add "Unload tab" option to right-click context menu (future feature)

### `tabGroups` - Read tab group information

**What it does**:

- Read tab group names and colors
- Display groups in popup
- Unload tabs by group

**What it does NOT do**:

- Modify or create tab groups
- Share group information

### Optional Host Permissions (v0.0.4+)

TabRest requests **optional** host permissions only when you enable certain features:

**`http://*/*` and `https://*/*` (Optional)**

**When requested**: Only if you enable one of these features:
- Form protection ("Protect tabs with unsaved forms" setting)
- Discarded tab title prefix (to prepend symbol to unloaded tab titles)

**What it does**:
- Inject form-detection script when form protection is enabled
- Inject title-update script when prefix setting is enabled

**What it does NOT do**:
- Runs only when the feature is enabled, not always
- Collect or store page content
- Access sensitive data

**How to control**:
- Enable/disable in Settings → "Protect tabs with unsaved forms" or "Show discarded prefix"
- If you disable the feature, you can revoke the permission in Chrome:
  1. Go to `chrome://extensions`
  2. Find TabRest → **Details**
  3. Scroll to **Host permissions**
  4. Click **Remove** if the feature is disabled
- Chrome will re-prompt if you re-enable the feature

**Why optional**: Only extensions that truly need host access request it. TabRest respects your privacy by asking only when needed.

TabRest does **not** use:

- ❌ Analytics services (Google Analytics, etc.)
- ❌ Error tracking (Sentry, Rollbar, etc.)
- ❌ Ad networks
- ❌ CDNs for loading code
- ❌ External APIs
- ❌ Any third-party services whatsoever

**All code runs locally in your browser.**

## Data Sharing

TabRest does **not** share any data with:

- Third-party companies
- Advertisers
- Analytics providers
- Other extensions
- Anyone

**Exception**: If you enable Chrome Sync, Chrome itself syncs settings and whitelist across your devices. We do not control or access this synced data.

## Chrome Sync

TabRest uses `chrome.storage.sync` for settings and whitelist. If Chrome Sync is enabled in your browser:

- **What syncs**: Settings and whitelist only
- **Who controls it**: Google Chrome, not TabRest
- **How to disable**: Turn off Chrome Sync in Chrome settings
- **Privacy**: Governed by [Google's Privacy Policy](https://policies.google.com/privacy), not ours

**We do not access or control your Chrome Sync data.**

## Data Security

Since TabRest:

- Stores data only locally
- Makes no network requests
- Contains no server-side components

**There is no data to breach.** Your data never leaves your device (except via Chrome Sync if enabled).

## Children's Privacy

TabRest does not collect any data from anyone, including children under 13 (or equivalent minimum age in your jurisdiction).

## Data Retention

- **Settings & whitelist**: Stored until you uninstall the extension or clear Chrome storage
- **Tab activity**: Stored until you uninstall the extension
- **Statistics**: Stored until you reset stats or uninstall the extension

**To delete all data**:

1. Go to `chrome://extensions`
2. Find TabRest → **Remove**
3. All data is deleted immediately
