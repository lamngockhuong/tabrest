# 标签组

TabRest 与 Chrome 原生的标签组功能集成，让您可以一次性卸载整个组。

## 什么是标签组？

标签组是 Chrome 的一项功能（在 Chrome 88 中加入），它让您可以将标签页组织成带标签、颜色编码的组。

**创建标签组**：
1. 右键标签页 → **添加到新组**
2. 选择名称和颜色
3. 将标签页拖入组中

**Chrome 的标签组**：
- 可折叠以节省空间
- 颜色编码，便于视觉整理
- 可一起移动
- 跨设备同步（如果启用了 Chrome 同步）

## 卸载标签组

TabRest 在弹出窗口中添加了基于组的卸载：

### 从弹出窗口

1. 点击工具栏中的 **TabRest 图标**
2. 滚动到 **标签组** 部分
3. 点击组名
4. 该组内的所有标签页都将被卸载

**视觉指示器**：
- 弹出窗口中显示组名和颜色
- 显示每个组中的标签页数量
- 已卸载的标签页标有 💤 图标

### 组内受保护的标签页

即使在卸载一个组时，单个标签页的保护仍然适用：

| 标签页类型 | 行为 |
|----------|----------|
| **固定标签页** | 受保护（如果"卸载固定标签页"被禁用） |
| **音频标签页** | 受保护（如果"保护音频标签页"已启用） |
| **表单标签页** | 受保护（如果"保护表单标签页"已启用） |
| **白名单站点** | 受保护（始终） |

**示例**：如果您卸载一个含有 10 个标签页的组，但其中 2 个含有未保存的表单，则只有 8 个会被卸载。

## 自动卸载

TabRest 的自动卸载（基于计时器和内存阈值）适用于所有标签页，包括分组的标签页。

**没有特殊的组保护**：标签组不会阻止自动卸载。如果您想保护某个组：

1. 将组的域名添加到白名单
2. 固定组中的标签页
3. 保持标签页活动（定期与它们交互）

## 标签组设置

TabRest 包含一个标签组功能的切换开关：

**启用标签组**（默认：启用）：
- 启用时：弹出窗口中显示标签组部分
- 禁用时：组从弹出窗口中隐藏，但自动卸载仍然有效

**要禁用**：
1. 打开设置页面
2. 关闭 **启用标签组** 切换
3. 更改自动保存

## 使用场景

### 按项目组织

为不同项目或场景创建组：

- **工作**：公司邮件、项目文档、问题跟踪器
- **个人**：社交媒体、银行、购物
- **研究**：文章、文档、教程

在切换场景时卸载整个项目。

### 会话管理

将组用作临时会话：

1. 为某个任务打开标签页
2. 将它们分组在一起
3. 完成后，卸载整个组
4. 保持标签页分组以供稍后使用

比书签更好，因为：
- 标签页保持打开（只是被卸载）
- 恢复更快
- 保留滚动位置和表单数据

### 内存管理

识别内存占用高的组：

1. 为内存密集型站点（例如视频、设计工具）创建组
2. 在不主动使用时卸载该组
3. 需要时重新加载

## 提示

### 命名组

使用描述性名称以便快速识别：

```
✅ 良好："Project X Documentation"
✅ 良好："Q1 Planning"
❌ 不佳："Group 1"
❌ 不佳："Tabs"
```

### 颜色编码

按优先级或类别分配颜色：

- 🔴 红色：紧急/正在进行的工作
- 🟢 绿色：参考/背景
- 🔵 蓝色：个人任务
- 🟡 黄色：等待/审核
- 🟣 紫色：研究/学习

### 与白名单结合

对于关键的组，将域名添加到白名单：

1. 识别组中的标签页
2. 将域名添加到白名单
3. 组会自动保持加载

## 局限性

### Chrome 专属功能

标签组是 Chrome 的功能。TabRest 需要：

- **Chrome 88+** 以支持标签组
- **Chrome 89+** 以在 API 中支持标签组颜色

**没有标签组的浏览器**：TabRest 的组功能会自动隐藏。

### 没有组级别的保护

TabRest 无法：
- 保护整个组不被自动卸载（请改为保护单个标签页）
- 创建或管理组（请使用 Chrome 的内置功能）
- 跨浏览器同步组（由 Chrome 同步处理）

### 扩展 API 局限性

TabRest 使用 `chrome.tabGroups` API，它：
- 无法修改组属性（名称、颜色）
- 无法创建或删除组
- 只能读取组信息并卸载标签页

## 键盘快捷键

TabRest 不为组操作提供键盘快捷键。请使用 Chrome 的内置快捷键：

| 操作 | Windows/Linux | macOS |
|--------|---------------|-------|
| 创建组 | `Ctrl+G` | `Cmd+G` |
| 下一组 | `Ctrl+Shift+9` | `Cmd+Shift+9` |
| 上一组 | `Ctrl+Shift+0` | `Cmd+Shift+0` |

**注意**：这些快捷键可能因 Chrome 版本而异。

## 故障排除

### 组未在弹出窗口中显示

**问题**：标签组部分为空。

**解决方案**：
1. **创建一个组**：TabRest 只显示存在的组
2. **检查设置**：确保"启用标签组"已启用
3. **重新加载扩展**：进入 `chrome://extensions` → TabRest → 刷新

### 组无法卸载

**问题**：点击组名没有任何反应。

**解决方案**：
1. **检查标签页保护**：某些标签页可能受保护（固定、音频、白名单）
2. **确认活动标签页**：您无法卸载当前活动的标签页
3. **检查 Chrome 版本**：需要 Chrome 88+

### 卸载后组消失

**问题**：卸载后组折叠或隐藏。

**说明**：这是 Chrome 的行为。折叠组中已卸载的标签页保持折叠状态。

**解决方案**：点击组标签以展开并查看已卸载的标签页。
