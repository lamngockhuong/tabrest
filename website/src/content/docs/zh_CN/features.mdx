# 功能

TabRest 提供多种方式来管理您的标签页并释放内存。

**弹出窗口 UI 提示**：弹出窗口中的可折叠部分（标签页、快速设置、更多操作、会话、统计信息）会在弹出窗口关闭后记住其展开/折叠状态，为您在重复操作时节省时间。

## 自动卸载

### 基于计时器的卸载

在一段不活动时间后自动卸载标签页：

| 计时器选项 | 使用场景 |
|--------------|----------|
| 5 分钟    | 最激进的内存节省 |
| 10 分钟   | 激进的内存节省 |
| 15 分钟   | 平衡偏激进 |
| 30 分钟   | 默认平衡模式 |
| 1 小时       | 轻度用户 |
| 2 小时      | 保留标签页更久 |
| 4 小时      | 最少的自动卸载 |

**工作原理**：TabRest 跟踪您上次与每个标签页交互的时间（点击、滚动、聚焦）。在配置的时间过后，不活动的标签页将被卸载。

### 内存阈值卸载

当系统 RAM 使用率超过阈值时卸载标签页：

- **范围**：60-95% 内存使用率
- **默认**：80%
- **检查间隔**：每 30 秒

触发时，TabRest 会卸载最近最少使用（LRU）的标签页，直到内存降至阈值以下。

### 启动时卸载

在 Chrome 启动时立即释放内存：

- **默认**：启用
- **行为**：在浏览器启动时强制卸载所有窗口中所有非活动标签页，绕过所有保护（固定、白名单、音频、表单）
- **适用于**：从上一会话恢复大量标签页而不浪费内存

### 标签页数量阈值

仅在您有足够多的不活动标签页时才开始自动卸载：

- **范围**：3-20 个标签页
- **默认**：6 个标签页
- **禁用**：设置为 0 则始终自动卸载

**使用场景**：防止在只有少量标签页打开时进行激进卸载。

### 仅在空闲状态

仅在您的计算机空闲时自动卸载标签页：

- **默认**：禁用
- **阈值**：2-15 分钟不活动
- **检测**：使用 Chrome 的 idle API（检测鼠标/键盘活动）

**使用场景**：绝不打断您正在进行的浏览会话。

### 单标签页内存阈值

自动卸载消耗过多内存的单个标签页：

- **范围**：100MB - 1GB（JavaScript 堆）
- **默认**：禁用
- **注意**：仅测量 JS 内存，而非标签页的总内存

**使用场景**：专门针对内存占用高的标签页，而非按时长处理。

#### 暂停警告提示

在自动卸载发生前看到 3 秒的页内警告：

- **默认**：启用
- **延迟**：可配置为 1-10 秒（默认 3 秒）
- **行为**：在自动暂停前显示一个通知栏；您可以通过以下方式取消：
  - 切换到该标签页
  - 在页面上播放音频
  - 修改表单字段
  - 对标签页进行小睡
- **原因**：防止丢失未保存的工作或打断重要任务

**示例**：您离开了一个含有未保存文本的标签页。TabRest 会在卸载前 3 秒显示警告，让您有时间保存或取消卸载。

## 离线时跳过

在网络不可用时防止标签页被卸载：

- **默认**：启用
- **行为**：当 `navigator.onLine` 为 false 时暂停自动卸载
- **原因**：已卸载的标签页在没有网络时无法重新加载，避免造成困扰

## 全局暂停

用一个开关临时禁用所有自动标签页卸载：

- **访问方式**：弹出窗口和侧边栏顶部附近的暂停栏
- **持续时间**：30 分钟、1 小时、2 小时，或"直到恢复"（无限期）
- **暂停期间**：暂停栏会显示 **恢复** 按钮以及剩余时间（例如"45 分钟后恢复"），无限期选项则显示"直到恢复"
- **阻止**：不活动计时器、内存阈值、单标签页内存阈值以及启动时自动卸载
- **不阻止**：手动卸载操作（当前/其他/左侧/右侧）、右键上下文菜单、键盘快捷键以及工具栏点击操作，这些始终有效
- **指示器**：暂停时工具栏徽章显示 ❚❚；弹出窗口标签列表中的单标签页倒计时被隐藏
- **持久性**：按设备本地存储（不跨设备同步）；定时暂停在到达截止时间时自动恢复，可在 service worker 重启后保留

**对比小睡**：小睡保护单个标签页或域名；全局暂停是一个可一次性停止所有自动卸载的开关。

**使用场景**：演示、跨多个标签页下载大量文件、游戏或调试，任何您暂时不希望标签页被卸载的时候。

## 手动控制

从弹出窗口按需控制标签页卸载：

| 操作 | 描述 |
|--------|-------------|
| **卸载当前标签页** | 卸载当前活动的标签页 |
| **卸载其他标签页** | 卸载除当前之外的所有标签页 |
| **卸载左侧** | 卸载当前标签页左侧的所有标签页 |
| **卸载右侧** | 卸载当前标签页右侧的所有标签页 |

通过工具栏中的 TabRest 图标或[键盘快捷键](/docs/keyboard-shortcuts)访问。

### 关闭重复标签页

自动关闭当前窗口中的重复标签页：

- **访问方式**：打开弹出窗口 → 更多操作 → 关闭重复项
- **行为**：按 URL 扫描所有标签页，保留最旧的标签页，关闭较新的重复项
- **使用场景**：快速清理意外打开的同一站点的多个标签页

**示例**：如果您有三个标签页打开到 `github.com/user/project`，TabRest 会关闭较新的两个并保留第一个。

### 标签页搜索

弹出窗口中标签页的实时搜索过滤：

- **访问方式**：打开弹出窗口 → 搜索输入框（切换显示）
- **行为**：按标签页标题和 URL 实时过滤（不区分大小写）
- **可组合**：与过滤标签（全部/休眠/小睡/受保护）配合使用
- **使用场景**：当您打开许多标签页时快速找到特定标签页

**示例**：输入 "github" 以过滤所有与 GitHub 相关的标签页。

### 工具栏点击操作

自定义点击扩展图标时发生的操作：

| 选项 | 行为 |
|--------|----------|
| **打开弹出窗口** | 默认，显示标签列表和快速设置 |
| **在侧边栏中打开** | 在浏览器的持久侧边栏中打开 TabRest（可选启用） |
| **卸载当前标签页** | 立即卸载当前标签页 |
| **卸载其他标签页** | 立即卸载所有其他标签页 |

**使用场景**：希望一键卸载或持久访问侧边栏而无需打开弹出窗口的高级用户。

## 电源模式

根据您的情况调整自动卸载的激进程度：

| 模式 | 计时器倍数 | 内存偏移 |
|------|------------------|---------------|
| **省电模式** | 0.5 倍（更快） | -10%（更低阈值） |
| **普通** | 1.0 倍（默认） | 0% |
| **性能模式** | 2.0 倍（更慢） | +10%（更高阈值） |

**示例**：如果您的计时器设置为 30 分钟：
- 省电模式：15 分钟
- 普通：30 分钟
- 性能模式：60 分钟

## 标签页保护

保护特定标签页不被自动卸载：

### 固定标签页
- **默认**：受保护，不被自动卸载
- **可配置**：在设置中启用"卸载固定标签页"以覆盖此行为

### 音频/视频标签页
- **默认**：播放媒体时受保护
- **检测**：检查 `tab.audible` 属性
- **可配置**：禁用"保护播放音频/视频的标签页"以允许卸载

### 表单标签页
- **默认**：表单含有未保存数据时受保护
- **检测**：内容脚本检查被修改的输入字段和文本区域
- **权限**：按需的主机权限（仅在您启用表单保护时请求）
- **可配置**：在设置中禁用"保护含有未保存表单的标签页"以允许卸载
- **注意**：适用于标准 HTML 表单；可能无法检测富文本编辑器（TinyMCE、CKEditor），对于这些站点请使用白名单

### 白名单
- **默认**：YouTube、Gmail、Google Docs、Miro、Figma、Notion
- **自定义**：添加任何站点以防止自动卸载
- **详情**：参见[白名单配置](/docs/whitelist-config)

### 小睡标签页/域名

临时保护特定标签页或整个域名不被自动卸载：

| 小睡选项 | 持续时间 |
|---------------|----------|
| 标签页小睡 | 30 分钟、1 小时、2 小时 |
| 域名小睡 | 1 小时 |

**访问方式**：在页面上右键 → 小睡选项，或使用弹出窗口标签列表中的 ⏸️ 按钮。

**使用场景**：密集地在某个站点上工作，而无需将其添加到永久白名单。

## 侧边栏模式

在 Chrome 的持久侧边栏中打开 TabRest，而非弹出窗口：

- **访问方式**：设置 → 工具栏操作 → 切换"在侧边栏中打开"
- **默认**：禁用（可选启用功能）
- **行为**：打开一个在切换标签页时保持可见的侧边栏面板
- **响应式**：面板会针对不同屏幕尺寸调整布局
- **恢复**：关闭切换即可返回弹出窗口模式
- **要求**：Chrome 114+ 以支持侧边栏 API

**使用场景**：始终保持 TabRest 可见，以便快速访问标签页控制和统计信息，而无需在操作之间关闭面板。

## 标签组

一次性卸载整个标签组：

1. **在 Chrome 中创建标签组**（右键标签页 → 添加到新组）
2. **打开 TabRest 弹出窗口** 并点击组名
3. **组内所有标签页** 一起卸载

**注意**：标签组是 Chrome 的功能（Chrome 88+）。TabRest 与现有的组集成。

## 导入与导出

备份并恢复您的配置：

- **导出内容**：白名单、黑名单和已保存的会话
- **格式**：JSON（基于剪贴板，易于存储/共享）
- **架构版本**：v1（面向未来的迁移）
- **导入行为**：追加式（新项目添加到现有列表，自动移除重复项）
- **访问方式**：设置或弹出窗口 → 会话部分 → 导出/导入按钮
- **使用场景**：在浏览器之间转移设置、在重大更改前备份、与团队成员共享白名单

**示例**：将您的白名单导出为 JSON，通过电子邮件发送，然后在另一台设备上导入。

## 统计信息

跟踪您随时间节省的内存：

- **已卸载标签页**：自安装以来的总数
- **已节省内存**：估算释放的 MB 数（根据标签页内存使用量计算）
- **查看统计**：打开设置页面
- **重置统计**：使用设置中的重置按钮

**隐私**：所有统计信息均本地存储。没有数据离开您的浏览器。

## 徽章指示器

在扩展图标上显示已卸载标签页的数量：

- **默认**：启用
- **显示**：工具栏图标上的小数字徽章
- **禁用**：在设置中关闭"显示徽章计数"

## RAM 使用量显示

在弹出窗口头部查看当前系统 RAM 使用量：

- **显示**：与标签页统计信息一起显示 RAM 百分比
- **警告**：当 RAM 超过您配置的阈值时高亮显示
- **内存估算**：将鼠标悬停在 RAM 统计上，查看每个已卸载标签页释放内存的估算值（平均每个标签页约 150 MB）
- **工具提示**：显示阈值对比和估算的内存恢复量

## 滚动位置恢复

在标签页重新加载时自动保存并恢复滚动位置：

- **默认**：启用
- **行为**：在卸载前保存位置，在重新加载时恢复
- **存储**：本地存储，自动清理旧条目（最多 100 条）

**使用场景**：从您上次停下的确切位置继续阅读长文章。

## 自动卸载通知

在标签页被自动卸载时收到通知：

- **默认**：禁用
- **内容**：显示标签页标题、原因（计时器/RAM）和详情
- **静音**：通知不播放声音

**使用场景**：随时了解 TabRest 在后台的操作。

## 以暂停状态打开链接

直接以暂停（已卸载）状态打开链接：

- **访问方式**：右键任意链接 → "在暂停的标签页中打开链接"
- **行为**：创建标签页，等待加载，然后立即卸载
- **使用场景**：保存链接以供稍后使用而不消耗内存

## 多语言支持

TabRest 支持：

- 🇺🇸 英语
- 🇻🇳 越南语

语言从 Chrome 设置自动检测。

## 技术功能

### Chrome Manifest V3
- 现代化的 service worker 架构
- 高效的资源使用
- 符合最新的 Chrome 标准

### 可选主机权限
- 仅在您启用表单保护或其他依赖主机的功能时请求权限
- 随时可在 Chrome 设置中撤销权限而不丢失其他功能
- 功能激活期间提供透明的权限提示

### 按需表单检测
- 仅在启用表单保护时注入表单检查脚本
- 禁用时对性能影响极小
- 检测被修改的输入字段、文本区域和下拉选择框

### ES 模块
- 原生 JavaScript 模块
- 无需构建步骤
- 快速加载和执行

### Chrome Alarms API
- 在 service worker 终止后仍能存续
- 可靠的周期性检查
- 省电的调度
