# 隐私政策

**最后更新**：2026 年 4 月 29 日

TabRest 致力于保护您的隐私。本政策说明我们收集（或不收集）哪些数据以及如何使用这些数据。

## 摘要

- ✅ **默认不收集任何个人数据**
- ✅ **默认无网络请求，所有数据都保留在您的设备上**
- ✅ **绝不进行跟踪、分析或行为遥测**
- ✅ **匿名崩溃报告完全为可选启用（默认关闭，参见[错误报告](#error-reporting)）**
- ✅ **开源且可审计**

## 数据收集

TabRest 将行为分为两种不同的模式。默认情况下，扩展完全在本地运行。有一项功能，即匿名崩溃报告，是可选启用的，仅在您明确启用后才向 Sentry 发送经过脱敏处理的负载。

### 我们收集什么

| 数据类型 | 默认情况下（开箱即用） | 如果您选择启用错误报告 |
| --- | --- | --- |
| 浏览历史（您访问的 URL） | ❌ 从不收集 | ❌ 从不收集（堆栈跟踪中的 URL 在客户端被替换为 `[REDACTED]`） |
| 页面内容、标题、表单数据 | ❌ 从不收集 | ❌ 从不收集 |
| 个人信息（姓名、电子邮件、IP） | ❌ 从不收集 | ❌ 电子邮件/IP 在客户端被脱敏；Sentry 负载设置 `user.ip_address: null` |
| 使用分析／行为遥测 | ❌ 从不收集 | ❌ 从不收集（无页面浏览、点击或功能使用跟踪） |
| 崩溃报告 | ❌ 从不收集 | ✅ 经过脱敏的错误名称、消息、堆栈跟踪 |
| 设备／环境 | ❌ 从不收集 | ✅ 扩展版本、浏览器 + 操作系统字符串 |
| 汇总计数器 | ❌ 从不收集 | ✅ 标签页数量（总计/已卸载/固定/有声）、RAM 统计、设置值、安装日期 |
| 白名单／黑名单内容 | ❌ 从不收集 | ❌ 排除，仅条目计数 |

可选启用的错误报告在[错误报告](#error-reporting)中有完整详细的说明。您可以随时在选项 → 隐私与诊断中禁用它。

### 我们在本地存储什么

TabRest **仅在您的设备上** 存储以下数据：

| 数据类型    | 存储位置       | 用途                                   | 是否同步？ |
| ------------ | ---------------------- | ----------------------------------------- | ------- |
| 设置     | `chrome.storage.sync`  | 用户偏好（计时器、阈值等） | 是\*   |
| 白名单    | `chrome.storage.sync`  | 受保护的域名                         | 是\*   |
| 标签页活动 | `chrome.storage.local` | 每个标签页的上次活动时间（用于 LRU）      | 否      |
| 统计信息   | `chrome.storage.local` | 已卸载标签页计数、已节省内存         | 否      |

\*如果您在浏览器设置中启用了 Chrome 同步，则通过 Chrome 同步进行同步。我们不控制或访问此同步数据。

## Chrome 权限

TabRest 需要以下权限：

### `tabs` - "读取您的浏览历史记录"

**它实际做什么**：

- 读取标签页 URL 以匹配白名单
- 读取标签页状态（固定、音频、活动）
- 调用 `chrome.tabs.discard()` 卸载标签页

**它不会做什么**：

- 将您的浏览历史记录发送到任何地方
- 跟踪您访问哪些站点
- 存储您的浏览历史记录

**为什么 Chrome 显示此标签**：`tabs` 权限提供对标签页 URL 的访问。Chrome 的权限标签是通用的，并不反映实际用途。

### `storage` - 在本地存储数据

**它做什么**：

- 保存设置（计时器、阈值、保护）
- 保存白名单条目
- 存储标签页活动以进行 LRU 跟踪
- 跟踪统计信息（已卸载标签页、已节省内存）

**所有数据都保留在您的设备上。** 无外部存储或云服务。

### `alarms` - 调度周期性任务

**它做什么**：

- 调度周期性标签页检查（每 1 分钟）
- 调度内存阈值检查（每 30 秒）

**为什么需要**：Chrome service worker 在空闲时终止。Alarms 会唤醒 service worker 以执行检查。

### `system.memory` - 读取内存使用量

**它做什么**：

- 读取系统总 RAM 和可用 RAM
- 计算内存使用百分比
- 在超过阈值时触发卸载

**它不会做什么**：

- 访问任何其他系统信息
- 收集或存储内存数据
- 将内存数据发送到任何地方

### `contextMenus` - 添加右键菜单项

**它做什么**：

- 向右键上下文菜单添加"卸载标签页"选项（未来功能）

### `tabGroups` - 读取标签组信息

**它做什么**：

- 读取标签组的名称和颜色
- 在弹出窗口中显示组
- 按组卸载标签页

**它不会做什么**：

- 修改或创建标签组
- 共享组信息

### 可选主机权限（v0.0.4+）

TabRest 仅在您启用某些功能时才请求 **可选的** 主机权限：

**`http://*/*` 和 `https://*/*`（可选）**

**何时请求**：仅当您启用以下功能之一时：
- 表单保护（"保护含有未保存表单的标签页"设置）
- 已卸载标签页的标题前缀（在已卸载标签页标题前添加符号）

**它做什么**：
- 在启用表单保护时注入表单检测脚本
- 在启用前缀设置时注入标题更新脚本

**它不会做什么**：
- 仅在功能启用时运行，并非始终运行
- 收集或存储页面内容
- 访问敏感数据

**如何控制**：
- 在设置中启用/禁用"保护含有未保存表单的标签页"或"显示已卸载前缀"
- 如果您禁用了该功能，可以在 Chrome 中撤销权限：
  1. 进入 `chrome://extensions`
  2. 找到 TabRest → **详细信息**
  3. 滚动到 **主机权限**
  4. 如果功能已禁用，点击 **移除**
- 如果您重新启用该功能，Chrome 会重新提示

**为什么是可选的**：只有真正需要主机访问权限的扩展才会请求它。TabRest 通过仅在需要时才请求来尊重您的隐私。

## 第三方服务

| 第三方服务 | 默认情况下 | 如果您选择启用 |
| --- | --- | --- |
| 分析（Google Analytics 等） | ❌ 从不使用 | ❌ 从不使用 |
| 广告网络 | ❌ 从不使用 | ❌ 从不使用 |
| 用于加载代码的 CDN | ❌ 从不使用（扩展代码已打包并在本地运行） | ❌ 从不使用 |
| 外部 API | ❌ 无 | ❌ 无 |
| 错误跟踪（Sentry SaaS，美国区域） | ❌ 已禁用，不发送请求 | ✅ 仅在您勾选可选启用开关后启用。参见[错误报告](#error-reporting)了解确切的负载。 |

**所有其他代码都在您的浏览器本地运行。** Sentry 端点（`*.ingest.us.sentry.io`）是 TabRest 唯一可能联系的网络目的地，且仅在您启用错误报告时。

## 数据共享

TabRest **不会** 与第三方公司、广告商、分析提供商或其他扩展共享任何数据。

**两个例外：**

1. **Chrome 同步**（如果您在 Chrome 中启用）：Chrome 本身会在您的设备之间同步设置和白名单。我们不控制或访问此同步数据。
2. **Sentry SaaS**（仅当您选择启用错误报告时）：经过脱敏的崩溃负载会被发送到 Sentry 的美国区域摄取端点。参见[错误报告](#error-reporting)了解完整的负载规格、保留期和退出说明。

## Chrome 同步

TabRest 使用 `chrome.storage.sync` 存储设置和白名单。如果您的浏览器启用了 Chrome 同步：

- **同步内容**：仅设置和白名单
- **由谁控制**：Google Chrome，而非 TabRest
- **如何禁用**：在 Chrome 设置中关闭 Chrome 同步
- **隐私**：受 [Google 隐私政策](https://policies.google.com/privacy) 约束，而非我们的

**我们不访问或控制您的 Chrome 同步数据。**

## 数据安全

默认情况下，TabRest：

- 仅在本地存储数据
- 不发出任何网络请求
- 不包含任何服务器端组件

**在禁用错误报告（默认）的情况下，没有可被泄露的数据。** 除非您启用 Chrome 同步，否则您的数据永远不会离开您的设备。如果您选择启用错误报告，经过脱敏的崩溃负载（无 URL、无 PII）会通过 HTTPS 传输到 Sentry 的美国区域摄取端点，并保留 30 天。

- **可审计**：任何人都可以审查代码
- **无混淆**：纯 JavaScript，无压缩或打包

## 儿童隐私

TabRest 不从任何人（包括 13 岁以下的儿童，或您所在司法管辖区的等效最低年龄）收集任何数据。

## 数据保留

- **设置和白名单**：存储至您卸载扩展或清除 Chrome 存储为止
- **标签页活动**：存储至您卸载扩展为止
- **统计信息**：存储至您重置统计或卸载扩展为止

**要删除所有数据**：

1. 进入 `chrome://extensions`
2. 找到 TabRest → **移除**
3. 所有数据立即被删除

## 错误报告

_通过 Sentry SaaS（美国区域）的崩溃报告。_

TabRest 可以发送匿名崩溃报告，以帮助更快地修复错误。**此功能默认关闭**，您必须通过选项 → 隐私与诊断选择启用。

**启用时发送的内容：**

- 经过脱敏的错误名称、消息和堆栈跟踪（URL、电子邮件地址和 IP 地址被替换为 `[REDACTED]`）
- 扩展版本、浏览器名称和版本、操作系统平台字符串
- 仅标签页计数：总计、已卸载、固定、有声以及窗口计数，无 URL 或页面标题
- 系统内存统计：使用百分比、总 RAM、可用 RAM
- 设置值（自动卸载计时器、RAM 阈值、功能开关等），白名单和黑名单域名被 **排除**，仅包含其条目计数
- 累计统计：所有时间和今日已暂停的标签页、安装日期

**绝不发送的内容：**

- 您访问的页面 URL
- 页面标题、内容、表单数据或滚动位置
- 电子邮件地址或 IP 地址（在传输前于客户端脱敏）
- 您的白名单或黑名单域名
- 任何可将报告关联到您的标识符

**手动错误报告：** 当您在弹出窗口的错误报告模态框中点击"发送到 Sentry"时，会包含一段经过脱敏的描述（已剥离 PII）以及上述诊断快照，仅此而已。

**第三方：** 报告被发送到 [Sentry](https://sentry.io)（美国区域，`*.ingest.us.sentry.io`）。参见 [Sentry 的隐私政策](https://sentry.io/privacy/) 了解其数据实践。

**您的 IP 地址：** Sentry 负载明确设置 `user.ip_address: null`，因此 Sentry 不会将您的 IP 与错误事件关联。

**数据保留：** Sentry 默认保留事件 30 天，之后自动删除。

**如何禁用：** 打开扩展的选项页面 → 隐私与诊断 → 取消勾选"启用匿名错误报告"。已发送的报告将在保留期到期时被删除，且无法撤回。

**自定义 DSN：** 高级用户可以在选项中提供自己的 Sentry DSN。无论目的地如何，都适用相同的脱敏规则。

## 法律

### 司法管辖

本扩展由个人开发者 Lam Ngoc Khuong 开发和维护。

### 合规

TabRest 遵守：

- Chrome 应用商店开发者计划政策
- 《通用数据保护条例》（GDPR），通过不收集任何数据实现
- 《加州消费者隐私法》（CCPA），通过不收集任何数据实现

### 您的权利

由于 TabRest 不收集任何个人数据：

- 没有可访问、修改或删除的数据
- 没有可导出的数据
- 没有可反对的数据处理活动

所有数据都本地存储在您的设备上，并完全由您控制。

## 透明度

TabRest 以透明为原则构建：

- **开源**：MIT 许可，公开仓库
- **无分析或行为遥测**：代码库中任何地方都没有页面浏览、点击或功能使用的跟踪代码
- **默认无外部请求**：在禁用报告的情况下审计网络流量，您将发现 TabRest 没有任何请求
- **一个可选启用的网络目的地**：仅 Sentry 的美国摄取端点，且仅在您启用错误报告时
- **简单架构**：原生 JavaScript，依赖极少，易于审计

**亲自验证**：克隆仓库并检查源代码。您将发现没有分析、没有跟踪，也没有文档记录的可选启用错误报告器之外的网络请求。

---

**简而言之**：TabRest 是一款隐私优先的扩展。默认情况下，您的数据保留在您的设备上，我们不收集任何内容、不发送任何内容、不跟踪任何内容。有一项功能，即错误报告，完全为可选启用且有清晰的文档说明。
