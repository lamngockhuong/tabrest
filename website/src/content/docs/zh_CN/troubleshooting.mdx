# 故障排除

使用 TabRest 时常见问题的解决方案。

## 标签页无法卸载

**问题**：点击"卸载"没有任何反应，或标签页拒绝卸载。

### 原因 1：标签页受保护

TabRest 默认保护某些标签页。请检查该标签页是否：

- **已固定**：固定标签页受保护（除非启用了"卸载固定标签页"）
- **正在播放音频**：含有活动音频的标签页受保护（除非禁用了"保护音频标签页"）
- **含有未保存的表单**：含有被修改输入字段的标签页受保护（除非禁用了"保护表单标签页"）
- **在白名单中**：该域名在您的白名单中

**解决方案**：在设置中禁用相关保护，或从白名单中移除该域名。

### 原因 2：标签页处于活动状态

Chrome 无法卸载当前活动（聚焦）的标签页。

**解决方案**：先切换到其他标签页，然后卸载目标标签页。

### 原因 3：Chrome 内部页面

Chrome 阻止扩展卸载内部页面：

- `chrome://` 页面（设置、扩展等）
- `chrome-extension://` 页面
- Chrome 应用商店页面
- 新标签页

**解决方案**：这些页面按设计无法被卸载。它们占用的内存极少。

### 原因 4：标签页刚加载

Chrome 可能会阻止卸载刚打开或重新加载的标签页（打开不足 5 秒）。

**解决方案**：等待几秒后再试。

### 原因 5：扩展权限

TabRest 需要 `tabs` 权限才能卸载标签页。

**解决方案**：
1. 进入 `chrome://extensions`
2. 找到 TabRest → **详细信息**
3. 确认权限包含"读取您的浏览历史记录"
4. 如果缺失，请移除并重新安装扩展

## 扩展无法工作

**问题**：TabRest 图标没有响应，或自动卸载未触发。

### 检查扩展状态

1. 进入 `chrome://extensions`
2. 在列表中找到 TabRest
3. 确保开关为 **开启**（蓝色）
4. 检查是否有错误提示（红色文字）

### 重新加载扩展

1. 进入 `chrome://extensions`
2. 找到 TabRest
3. 点击 **刷新图标**（↻）
4. 再次测试功能

### 检查 Service Worker

1. 进入 `chrome://extensions`
2. 找到 TabRest → **详细信息**
3. 滚动到 **检查视图**
4. 点击 **service worker**
5. 检查控制台是否有错误

**常见错误**：
- `Storage quota exceeded`：清除扩展存储
- `Alarm not registered`：重新加载扩展
- `Permission denied`：重新安装扩展

### 重新安装扩展

如果重新加载没有帮助：

1. **导出设置**（可选）：记下您的白名单和设置
2. 进入 `chrome://extensions`
3. 找到 TabRest → **移除**
4. 从 Chrome 应用商店或源代码重新安装
5. 恢复设置

## 键盘快捷键不起作用

**问题**：按下快捷键没有任何反应。

### 确认快捷键已设置

1. 进入 `chrome://extensions/shortcuts`
2. 找到 TabRest 部分
3. 确保快捷键已分配（不为空）

**解决方案**：如果为空，请手动设置快捷键。

### 检查冲突

快捷键可能与以下项冲突：
- 其他扩展
- Chrome 浏览器快捷键
- 操作系统快捷键

**解决方案**：
1. 进入 `chrome://extensions/shortcuts`
2. 如果快捷键显示"已被 Chrome 使用"，请选择其他组合
3. 禁用冲突扩展的快捷键

### 在不同环境中测试

某些快捷键在以下页面上不起作用：
- Chrome 内部页面（`chrome://`）
- 扩展页面（`chrome-extension://`）
- Chrome 应用商店

**解决方案**：在普通网页上测试（例如 `example.com`）。

### 重新加载扩展

1. 进入 `chrome://extensions`
2. 找到 TabRest → 刷新图标
3. 再次测试快捷键

## 内存使用率高

**问题**：尽管使用了 TabRest，Chrome 仍占用大量内存。

### 确认标签页确实在卸载

1. 打开任务管理器：`Shift+Esc`（Windows/Linux）或 Chrome 菜单 → 更多工具 → 任务管理器
2. 查找标记为"已卸载"或内存极低（< 1 MB）的标签页
3. 如果标签页显示高内存，则它们尚未被卸载

**解决方案**：检查标签页保护设置（固定、音频、表单、白名单）。

### 检查自动卸载设置

1. 打开 TabRest 设置
2. 确认 **计时器** 已启用（未设置为"从不"）
3. 确认 **内存阈值** 合理（60-95%）
4. 启用 **启动时卸载** 以在浏览器重启时释放内存

### 扩展和后台页面

TabRest 无法卸载：
- 扩展页面本身
- 来自其他扩展的 service worker
- Chrome 自身的进程

**解决方案**：使用 Chrome 任务管理器（`Shift+Esc`）识别内存占用高的扩展并禁用它们。

### 标签页过多

如果您打开了数百个标签页：
- 自动卸载需要时间来处理所有标签页
- 某些标签页可能受保护或最近处于活动状态
- 考虑关闭您不再需要的标签页

**解决方案**：使用"卸载其他标签页"立即手动卸载所有不活动的标签页。

## 统计信息未更新

**问题**：统计页面显示 0 个已卸载标签页或过时的数字。

### 检查统计开关

1. 打开设置
2. 确保 **启用统计** 为开启
3. 如果之前关闭，请将其打开（统计将从该时刻开始跟踪）

### 确认卸载正在发生

1. 使用弹出窗口手动卸载一个标签页
2. 打开设置 → 统计部分
3. 检查计数是否增加

**解决方案**：如果统计仍未更新，请重新加载扩展。

### 重置统计

如果统计已损坏：

1. 打开设置
2. 点击 **重置统计** 按钮
3. 确认操作
4. 统计将从零开始跟踪

## 白名单不起作用

**问题**：白名单站点仍被卸载。

### 确认域名格式

白名单条目应仅为域名：

```
✅ 正确：github.com
✅ 正确：mail.google.com
❌ 错误：https://github.com
❌ 错误：github.com/user/repo
```

**解决方案**：从条目中移除协议（`https://`）和路径（`/user/repo`）。

### 检查黑名单模式

如果启用了 **黑名单模式**，行为将反转：
- 白名单变为黑名单（要卸载的站点）
- 所有其他站点均受保护

**解决方案**：如果您想要正常的白名单行为，请在设置中禁用黑名单模式。

### 大小写敏感性

域名匹配不区分大小写：
- `GitHub.com` = `github.com` ✅

**解决方案**：使用小写以保持一致。

### Chrome 内存节省程序也处于活动状态

**在 Chrome 108+ 上白名单看似失效时最常见的原因。**

Chrome 自带其自己的标签页卸载功能（`chrome://settings/performance` → 内存节省程序）。它运行在浏览器层面并忽略扩展状态。如果内存节省程序已启用，即使域名在 TabRest 的白名单中，它也可能卸载该标签页，因为内存节省程序从不咨询 TabRest。

**如何确认这是原因**：

1. 标签页在没有先出现 TabRest 暂停警告提示的情况下就卸载了
2. 即使 TabRest 处于暂停或禁用状态，标签页仍被卸载
3. 您在 `chrome://settings/performance` 中看到内存节省程序已开启

**解决方案（择一）**：

- **方案 A**：打开 `chrome://settings/performance` 并关闭内存节省程序。TabRest 随后将拥有完全控制权。
- **方案 B**：保持内存节省程序开启，但同时在 `chrome://settings/performance` 的 **始终保持这些网站处于活动状态** 列表中添加该域名。您需要手动保持 TabRest 白名单和 Chrome 列表同步。

参见[白名单配置 → 与 Chrome 内存节省程序共存](/docs/whitelist-config#coexistence-with-chrome-memory-saver)了解完整说明。

## 表单保护不起作用

**问题**：含有未保存表单的标签页被卸载。

### 检查表单保护设置

1. 打开设置
2. 确保 **保护含有未保存表单的标签页** 已启用

### 确认表单检测

TabRest 检测含有以下项的表单：
- 被修改的 `<input>` 字段（文本、电子邮件、密码等）
- 被修改的 `<textarea>` 字段
- 被修改的 `<select>` 下拉框

**不检测**：
- 富文本编辑器（例如 TinyMCE、CKEditor），请改用白名单
- 基于 Canvas 的编辑器
- 自定义 JavaScript 表单库

**解决方案**：将含有复杂表单的站点添加到白名单。

### 内容脚本时机

如果表单在页面加载后很快被填写，TabRest 可能尚未检测到它。

**解决方案**：填写表单后等待几秒再切换标签页。

## 侧边栏问题

### 侧边栏无法打开

**问题**：点击工具栏图标打开的是弹出窗口而非侧边栏，或侧边栏切换不可用。

**可能原因**：

1. **浏览器过旧**：侧边栏 API 需要 Chrome 114+
2. **设置未启用**：检查设置中"在侧边栏中打开"切换是否为开启

**解决方案**：
- 将 Chrome 更新到 114 或更高版本
- 打开设置并启用侧边栏切换
- 重新加载扩展（`chrome://extensions` → 刷新）

### 侧边栏关闭或消失

**问题**：切换标签页时侧边栏关闭，或意外消失。

**可能原因**：

1. **面板已折叠**：检查面板是否在边缘处折叠
2. **Service worker 崩溃**：重新加载扩展

**解决方案**：
1. 如果面板在边缘处，点击以展开
2. 重新加载扩展：进入 `chrome://extensions` → TabRest → 刷新
3. 关闭"在侧边栏中打开"再重新开启

## 表单保护问题

### 表单保护请求权限

**问题**：启用"保护含有未保存表单的标签页"会提示请求主机权限。

**说明**：表单保护需要向网页注入检测脚本。这需要可选的主机权限。

**解决方案**：
1. 当 Chrome 提示时点击 **允许**
2. 如果您不小心点击了 **阻止**，可以恢复：
   - 进入 `chrome://extensions`
   - 找到 TabRest → **详细信息**
   - 滚动到 **主机权限**
   - 点击 **允许** 或更改为"所有站点"

### 暂停警告未出现

**问题**：自动暂停前没有显示警告提示。

**可能原因**：

1. **未授予主机权限**：暂停警告注入需要主机访问权限
2. **设置被禁用**：检查"显示暂停警告"是否已启用
3. **Service worker 崩溃**：重新加载扩展

**解决方案**：
1. 打开设置并启用"显示暂停警告"
2. 检查主机权限（参见上文"表单保护请求权限"）
3. 重新加载扩展：`chrome://extensions` → TabRest → 刷新

## 报告错误

如果您遇到此处未列出的错误：

### 收集信息

1. **Chrome 版本**：`chrome://version`
2. **TabRest 版本**：`chrome://extensions` → TabRest → 详细信息
3. **重现步骤**：哪些操作触发了错误
4. **预期行为**：应该发生什么
5. **实际行为**：实际发生了什么
6. **控制台错误**：打开 service worker 检查器，检查错误

### 在 GitHub 上报告

1. 访问 [TabRest Issues](https://github.com/lamngockhuong/tabrest/issues)
2. 搜索现有 issue 以避免重复
3. 点击 **New Issue**
4. 包含上述所有信息
5. 如有帮助，请添加截图

### 紧急应对方法

如果 TabRest 导致严重问题：

1. 禁用扩展：`chrome://extensions` → 切换为关闭
2. 在 GitHub 上报告错误
3. 等待修复或从源代码重新安装
