# 白名单配置

TabRest 允许您使用白名单（或反向模式下的黑名单）保护特定网站不被自动卸载。

## 默认白名单

TabRest 预先配置了以下站点：

- `youtube.com` - 保护视频和播放列表
- `gmail.com` - 保持邮件标签页加载
- `docs.google.com` - 保护 Google 文档、表格、幻灯片

这些可以在设置中移除或自定义。

## 将站点添加到白名单

### 从选项页面

1. 点击 TabRest 图标 → **⚙️ 设置** 按钮
2. 滚动到 **白名单** 部分
3. 输入域名（例如 `github.com`）
4. 按 **Enter** 或点击 **添加**
5. 更改自动保存

### 从弹出窗口快速添加

直接从弹出窗口快速添加当前站点：

1. 导航到您想要保护的站点
2. 点击工具栏中的 TabRest 图标
3. 查找 **将当前站点添加到白名单** 链接
4. 点击以添加当前域名

**注意**：此功能仅添加域名（例如 `example.com`），而非完整 URL。

## 域名匹配规则

TabRest 使用简单的域名匹配：

### 精确域名匹配

```
白名单：github.com
✅ 匹配：https://github.com/user/repo
✅ 匹配：http://github.com/explore
❌ 不匹配：https://gist.github.com
❌ 不匹配：https://api.github.com
```

### 子域名匹配

要保护所有子域名，请添加基础域名：

```
白名单：google.com
✅ 匹配：docs.google.com
✅ 匹配：mail.google.com
✅ 匹配：drive.google.com
```

**或者** 添加特定子域名：

```
白名单：docs.google.com
✅ 匹配：https://docs.google.com/document/...
❌ 不匹配：https://mail.google.com
```

### 常见模式

| 白名单条目 | 保护范围 |
|-----------------|----------|
| `example.com` | `example.com` 及所有子域名 |
| `www.example.com` | 仅 `www.example.com` |
| `app.example.com` | 仅 `app.example.com` |

## 移除站点

1. 打开 **设置** 页面
2. 在白名单中找到该域名
3. 点击它旁边的 **X** 按钮
4. 更改自动保存

## 黑名单模式

反转白名单行为：不再保护列出的站点，而是**仅**卸载列出的站点。

### 如何启用

1. 打开 **设置** 页面
2. 切换 **黑名单模式** 开关
3. 除列表中的站点外，所有站点都将受保护

### 使用场景

**白名单模式（默认）**：保护少数重要站点，卸载其他所有内容

```
白名单：gmail.com, docs.google.com
→ 仅这些站点受保护
→ 所有其他站点均可被卸载
```

**黑名单模式**：保护大多数站点，激进地卸载特定站点

```
黑名单：news.com, social.com
→ 仅这些站点可被卸载
→ 所有其他站点均受保护
```

**何时使用黑名单模式**：
- 您希望默认保护大多数站点
- 您只想卸载少数几个内存占用高的站点（例如新闻站点、社交媒体）
- 您有一份简短的待卸载站点列表

## 白名单 + 其他保护

白名单与其他标签页保护配合使用：

| 保护类型 | 优先级 | 描述 |
|-----------------|----------|-------------|
| 白名单 | 最高 | 基于域名的保护 |
| 固定标签页 | 高 | 保护固定标签页（可配置） |
| 音频标签页 | 高 | 保护播放媒体的标签页（可配置） |
| 表单标签页 | 高 | 保护含有未保存表单的标签页（可配置） |

**示例**：即使某个站点未被列入白名单，在以下情况下它也不会被卸载：
- 标签页已固定（且"卸载固定标签页"被禁用）
- 标签页正在播放音频（且"保护音频标签页"已启用）
- 标签页含有未保存的表单数据（且"保护表单标签页"已启用）

## 导入/导出

TabRest 支持通过 JSON 轻松备份和恢复白名单、黑名单和会话：

### 导出白名单/会话

1. 点击 TabRest 图标 → **⚙️ 设置**（或弹出窗口 → 会话部分）
2. 滚动到 **白名单** 或 **会话** 部分
3. 点击 **导出** 按钮
4. JSON 被复制到剪贴板
5. 粘贴到文本文件以保存

**导出格式**（v1 架构）：
```json
{
  "version": 1,
  "whitelist": ["github.com", "localhost"],
  "blacklist": [],
  "sessions": [...]
}
```

### 导入白名单/会话

1. 将您保存的 JSON 复制到剪贴板
2. 点击 TabRest 图标 → **⚙️ 设置**（或弹出窗口 → 会话部分）
3. 点击 **导入** 按钮
4. 从剪贴板粘贴 JSON
5. 点击 **确认**
6. 新项目将添加到您现有的列表中
7. 重复项会被自动移除

**导入行为**：
- **追加式**：新条目将添加到您现有的白名单/黑名单
- **去重**：如果您导入 `["github.com"]` 而它已存在，则不会重复
- **非破坏性**：不会删除现有条目

### 自动同步

白名单也存储在 `chrome.storage.sync` 中，如果您启用了 Chrome Sync，它会在您的各个 Chrome 浏览器之间自动同步。

**要禁用自动同步**：
1. 进入 Chrome 设置 → **同步和 Google 服务**
2. 关闭 **同步所有内容** 或取消勾选 **扩展程序**
3. 白名单将仅保留在此设备上

### 手动备份（高级）

对于偏好控制台访问的技术用户：

1. 打开 `chrome://extensions`
2. 找到 TabRest → **详细信息**
3. 滚动到 **检查视图：service worker**
4. 在控制台中运行：
   ```javascript
   chrome.storage.sync.get('settings', (data) => {
     console.log(JSON.stringify(data.settings.whitelist));
   });
   ```
5. 复制 JSON 数组输出

## 提示

### 管理大型白名单

- **具体化**：仅添加您确实希望保持加载的站点
- **定期检查**：移除您不再使用的站点
- **考虑黑名单模式**：如果您的白名单变得庞大，请切换到黑名单模式

### 常见白名单示例

**开发工作流**：
```
localhost
127.0.0.1
github.com
stackoverflow.com
```

**效率工具**：
```
mail.google.com
calendar.google.com
slack.com
notion.so
```

**通讯应用**：
```
discord.com
zoom.us
teams.microsoft.com
```

### 通配符支持

TabRest 目前**不**支持通配符（例如 `*.google.com`）。请分别添加每个域名：

```
✅ 添加：mail.google.com, drive.google.com, docs.google.com
❌ 不支持：*.google.com
```

## 与 Chrome 内存节省程序共存

自 Chrome 108 起，Chrome 自带了名为 **内存节省程序**（Memory Saver）的标签页卸载功能（位于 `chrome://settings/performance`）。它运行在浏览器层面，**不会**咨询任何扩展，包括 TabRest。

### 这意味着什么

TabRest 的白名单是告诉 **TabRest** 不要卸载某个域名。它**不会**告诉 **Chrome** 保持该标签页加载。如果 Chrome 内存节省程序已启用并判定某个标签页已不活动足够长时间，即使该域名在 TabRest 的白名单中，它也会卸载该标签页。

这是 Chrome 平台的限制。不存在能让标签页退出内存节省程序的扩展 API。

### 此情况的症状

- 域名在您的 TabRest 白名单中
- 标签页仍被自动卸载
- 卸载在没有 TabRest 暂停警告提示的情况下发生（因为不是 TabRest 所为）

### 如何完全保护一个标签页

选择**其中一个**：

**方案 A（推荐）：关闭 Chrome 内存节省程序**

1. 打开 `chrome://settings/performance`
2. 关闭 **内存节省程序** 开关
3. TabRest 现在拥有完全控制权；您的白名单将按预期工作

**方案 B：在 Chrome 内维护第二份列表**

1. 打开 `chrome://settings/performance`
2. 在 **始终保持这些网站处于活动状态** 下，添加您在 TabRest 白名单中拥有的相同域名
3. 两个系统都将跳过这些标签页

**自然受保护的标签页**（在两个系统中都无需配置即可生效）：

- 播放音频或视频的标签页
- 使用摄像头或麦克风的标签页
- 含有未保存表单输入的标签页
- 固定标签页

## 隐私

白名单条目：
- 本地存储在 Chrome 同步存储中
- 在您的各个 Chrome 浏览器之间同步（如果已启用同步）
- 绝不发送到外部服务器
- 绝不与第三方共享

参见[隐私政策](/docs/privacy-policy)了解详情。
