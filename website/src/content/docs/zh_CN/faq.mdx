# 常见问题

关于 TabRest 和标签页卸载的常见问题。

## 一般问题

### "卸载"是什么意思？

**卸载** 是指将标签页从内存中移除，同时保持它在标签栏中可见。标签页会显示其标题和网站图标，但页面内容在您点击之前不会加载。

这不同于 **关闭** 标签页，关闭会将其从标签栏中完全移除。

### 卸载和关闭标签页是一回事吗？

不是。主要区别：

| 特性 | 已卸载的标签页 | 已关闭的标签页 |
|---------|--------------|------------|
| 在标签栏中可见 | ✅ 是 | ❌ 否 |
| 内存使用 | 极低（约 1 MB） | 无 |
| 恢复方式 | 点击重新加载 | 从历史记录重新打开 |
| 保留位置 | ✅ 是 | ❌ 否 |
| 保留表单数据 | ✅ 是 | ❌ 否 |

### TabRest 能节省多少内存？

节省的内存取决于您使用的网站：

- **简单页面**：每个标签页 50-100 MB
- **复杂的网络应用**：每个标签页 200-500 MB
- **重型站点**（视频、设计工具）：每个标签页 500+ MB

**示例**：卸载 20 个典型标签页可释放 2-4 GB 的 RAM。

### Chrome 关闭时 TabRest 还能工作吗？

不能。TabRest 仅在 Chrome 打开时运行。不过：

- **启动时卸载**（默认启用）会在 Chrome 重启时卸载标签页
- **设置持久保留** 于浏览器会话之间
- **白名单同步** 到其他设备（如果启用了 Chrome 同步）

### TabRest 能离线工作吗？

可以。TabRest 完全能离线工作，因为：

- 所有逻辑在浏览器本地运行
- 不需要网络请求
- 不涉及外部服务器

**注意**：如果页面未被缓存，重新加载已卸载的标签页需要网络连接。

## 隐私与安全

### TabRest 会收集我的数据吗？

**不会。** TabRest 不收集任何个人数据。

- 无跟踪或分析
- 无数据发送到外部服务器
- 无遥测或崩溃报告

参见[隐私政策](/docs/privacy-policy)了解详情。

### TabRest 存储哪些数据？

TabRest 在 Chrome 中本地存储：

| 数据类型 | 存储 | 用途 |
|-----------|---------|---------|
| 设置 | `chrome.storage.sync` | 用户偏好 |
| 白名单 | `chrome.storage.sync` | 受保护的域名 |
| 标签页活动 | `chrome.storage.local` | LRU 跟踪 |
| 统计信息 | `chrome.storage.local` | 使用统计 |

**所有数据都保留在您的设备上**，并通过 Chrome 同步进行同步（如果启用）。

### 为什么 TabRest 需要"读取您的浏览历史记录"权限？

此权限（`tabs`）是以下操作所必需的：

- 读取标签页标题和 URL（用于白名单匹配）
- 检测标签页状态（固定、音频、活动）
- 通过 `chrome.tabs.discard()` 卸载标签页

**TabRest 不会**：
- 将您的浏览历史记录发送到任何地方
- 跟踪您访问哪些站点
- 与第三方共享数据

Chrome 的权限标签是通用的，并不反映实际用途。

### TabRest 是开源的吗？

是的。TabRest 采用 MIT 许可证，且[在 GitHub 上可用](https://github.com/lamngockhuong/tabrest)。

您可以：
- 审计源代码
- 报告问题
- 贡献改进
- Fork 以供自己使用

## 技术问题

### 什么是 Chrome Manifest V3？

Manifest V3（MV3）是 Chrome 最新的扩展平台。与 V2 的主要区别：

- **Service worker** 取代后台页面（更高效）
- **更严格的权限**（更好的安全性）
- **更好的性能**（Chrome 优先考虑 MV3 扩展）

TabRest 采用 MV3 构建，面向现代 Chrome 浏览器。

### TabRest 会拖慢我的浏览器吗？

不会。TabRest 的设计将性能影响降至最低：

- **Service worker**：仅在需要时运行，空闲时终止
- **周期性检查**：每分钟一次（标签页），每 30 秒一次（内存）
- **高效的 API**：使用原生 Chrome API，无繁重计算

**证明**：查看 Chrome 任务管理器（`Shift+Esc`），TabRest 通常使用 < 10 MB。

### TabRest 卸载不活动标签页的速度有多快？

最短的自动卸载计时器为 **5 分钟**。可用预设：5、10、15、30 分钟；1、2、4 小时。

**理由**：
- 防止打断正在进行的工作流
- 给后台任务完成的时间
- 在内存节省与可用性之间取得平衡

**替代方案**：使用手动卸载按钮以立即卸载。

### 为什么 TabRest 无法卸载 Chrome 内部页面？

Chrome 的安全模型阻止扩展修改：

- `chrome://` 页面（设置、扩展等）
- `chrome-extension://` 页面
- Chrome 应用商店页面

这可保护 Chrome 的核心功能免受恶意扩展的影响。

### TabRest 能在其他 Chromium 浏览器上工作吗？

TabRest 应该能在支持 Manifest V3 的基于 Chromium 的浏览器上工作：

- ✅ Google Chrome 88+
- ✅ Microsoft Edge 88+
- ✅ Brave 88+
- ✅ Opera 74+
- ❌ Firefox（不同的扩展 API）
- ❌ Safari（不同的扩展 API）

**注意**：官方支持仅针对 Google Chrome。其他浏览器可能可以工作，但未经测试。

## 功能问题

### 如何启用侧边栏模式？

TabRest 可以在持久侧边栏中打开，而非弹出窗口：

1. 点击 TabRest 图标 → **⚙️ 设置** 按钮
2. 滚动到 **工具栏操作** 部分
3. 将 **在侧边栏中打开** 切换为开启
4. 下次点击将在侧边栏中打开 TabRest，而非弹出窗口
5. 当您在标签页之间切换时，面板保持打开

**注意**：需要 Chrome 114+。如果不可用，说明您的浏览器过旧。

### 如何关闭重复标签页？

使用内置的重复标签页关闭器：

1. 打开 TabRest 弹出窗口
2. 滚动到 **更多操作** 部分
3. 点击 **关闭重复项**
4. TabRest 扫描当前窗口并关闭较新的重复标签页，保留最旧的那个

**示例**：如果您不小心打开了 `github.com` 三次，TabRest 会关闭标签页 #2 和 #3，保留标签页 #1。

### 如何备份我的白名单或会话？

将您的配置导出为 JSON 并保存/共享：

1. 点击 TabRest 图标 → **⚙️ 设置**（或打开弹出窗口 → 会话部分）
2. 滚动到 **白名单** 或 **会话** 部分
3. 点击 **导出**
4. JSON 被复制到剪贴板
5. 将其保存到文件或通过电子邮件发送

要在另一台设备上恢复：
1. 点击 **导入**
2. 从剪贴板粘贴 JSON
3. 新项目将添加到您现有的列表中（重复项会自动移除）

### 什么是暂停警告提示？

一个在 TabRest 自动卸载标签页之前出现的 3 秒页内通知：

- **显示内容**：页面上显示"将在 3 秒后暂停标签页……"消息
- **原因**：给您一个机会保存工作或取消卸载
- **如何取消**：
  - 点击该标签页（将焦点切换到它）
  - 在页面上播放音频
  - 编辑表单字段
  - 对标签页进行小睡
- **可自定义**：在设置中调整延迟（1-10 秒）

**示例**：您离开浏览器，一个标签页即将因内存原因被卸载。警告出现，给您 3 秒时间在您仍在使用它时取消。

### 如何临时阻止 TabRest 卸载标签页？

使用 **全局暂停**：

1. 打开 TabRest 弹出窗口（或侧边栏）
2. 在顶部附近找到暂停栏
3. 选择持续时间：**30 分钟**、**1 小时**、**2 小时** 或 **直到恢复**
4. TabRest 停止所有自动卸载（不活动计时器、内存阈值、单标签页内存阈值以及启动时自动卸载），直到暂停结束
5. 随时点击 **恢复** 即可提前重新开启自动卸载

**暂停期间仍有效的功能**：手动卸载操作（卸载当前/其他/左侧/右侧）、右键上下文菜单、键盘快捷键以及工具栏点击操作都继续工作，只有自动卸载被暂停。

**指示器**：暂停时工具栏徽章显示 ❚❚，且弹出窗口标签列表中的单标签页倒计时被隐藏。

**这与小睡有何不同？** 小睡保护一个特定的标签页或域名。全局暂停是一个可一次性停止所有地方自动卸载的单一开关。

### TabRest 能在打开 X 个标签页后自动卸载吗？

目前不能。TabRest 基于以下条件触发：
- 距上次活动的时间
- 系统内存阈值

**变通方法**：设置较低的计时器（5 分钟）或较低的内存阈值（60%）以进行激进卸载。

### TabRest 能仅卸载特定网站的标签页吗？

可以，使用 **黑名单模式**：

1. 打开设置
2. 启用 **黑名单模式**
3. 将域名添加到列表
4. 只有这些域名会被卸载；所有其他域名均受保护

参见[白名单配置](/docs/whitelist-config)了解详情。

### TabRest 是否支持在白名单中使用 localhost 和 IP 地址？

支持。您可以将本地开发服务器加入白名单：

```
localhost
127.0.0.1
192.168.1.100
::1 (IPv6 环回)
```

**使用场景**：保护您的本地开发环境不被自动卸载，同时让其他站点被卸载。

### TabRest 能一次性恢复所有已卸载的标签页吗？

不能。您必须逐个点击每个已卸载的标签页以重新加载它。

**理由**：一次性重新加载所有标签页会违背节省内存的初衷。

**变通方法**：使用 Chrome 的"将所有标签页加入书签"来保存一个会话，然后按需重新打开标签页。

### TabRest 能定时卸载吗（例如每天晚上 9 点）？

目前不能。TabRest 的自动卸载是持续性的，而非定时的。

**功能请求**：基于计划的卸载已列入未来版本的规划。请在 [GitHub Issues](https://github.com/lamngockhuong/tabrest/issues) 上跟踪。

### TabRest 能卸载其他窗口中的标签页吗？

可以。TabRest 可跨所有 Chrome 窗口工作：

- 自动卸载会检查所有窗口
- 从弹出窗口进行的手动卸载仅影响当前窗口
- 键盘快捷键仅影响当前窗口

### TabRest 能与标签页休眠扩展一起工作吗？

TabRest 应该能与其他标签页管理扩展一起工作，但：

- **可能存在冲突**：两个扩展可能会竞相卸载标签页
- **功能重复**：您可能不需要两个都用

**建议**：一次只使用一个标签页管理扩展。

### TabRest 如何与 Chrome 内置的内存节省程序交互？

Chrome 108+ 在 `chrome://settings/performance` 中提供 **内存节省程序**，它在浏览器层面卸载不活动的标签页。TabRest 和内存节省程序独立运行：

- **内存节省程序不会咨询 TabRest**。不存在能让标签页退出内存节省程序的 Chrome 扩展 API。
- **TabRest 的白名单和小睡仅适用于 TabRest 自己的卸载流程**。它们无法阻止 Chrome 卸载同一个标签页。

**如果两者都启用**，被列入白名单的标签页仍可能被 Chrome 卸载。要防止这种情况，请：

1. 关闭 Chrome 内存节省程序，让 TabRest 管理一切（推荐给希望进行精细控制的用户），或
2. 保持两者开启，同时在 `chrome://settings/performance` 的 **始终保持这些网站处于活动状态** 列表中添加关键域名

参见[白名单配置](/docs/whitelist-config#coexistence-with-chrome-memory-saver)了解详情。

### 为什么我的白名单标签页仍被卸载？

最常见的原因是 Chrome 内存节省程序与 TabRest 并行运行。TabRest 的白名单不影响 Chrome 的内存节省程序。请参见上面的问题了解修复方法。

其他可能的原因：

- 域名格式错误（使用 `example.com`，而非 `https://example.com/path`）
- 启用了黑名单模式（白名单变为卸载列表）
- 还安装了另一个标签页管理扩展

参见[故障排除 → 白名单不起作用](/docs/troubleshooting#whitelist-not-working)了解完整的诊断步骤。

## 故障排除

### 为什么有些标签页在卸载后立即重新加载？

**可能原因**：

1. **网站自动重新加载**：某些站点使用 JavaScript 检测卸载并强制重新加载
2. **扩展冲突**：另一个扩展可能正在重新加载标签页
3. **Chrome 同步**：标签页同步可能会从另一台设备恢复标签页

**解决方案**：将这些站点添加到白名单以防止卸载。

### 为什么徽章计数显示错误的数字？

**可能原因**：

1. **徽章已禁用**：检查设置 → "显示徽章计数"是否已启用
2. **Service worker 已终止**：徽章在 service worker 运行时更新
3. **扩展错误**：重新加载扩展

**解决方案**：打开弹出窗口（这会唤醒 service worker），徽章应会更新。

### 我能导出我的设置吗？

无法直接从 UI 导出。设置存储在 `chrome.storage.sync` 中，并通过 Chrome 同步自动同步。

**手动导出**（高级）：
1. 进入 `chrome://extensions`
2. TabRest → **详细信息** → **检查视图：service worker**
3. 在控制台中：
   ```javascript
   chrome.storage.sync.get('settings', console.log);
   ```
4. 复制输出

### 如何报告错误？

参见[故障排除 → 报告错误](/docs/troubleshooting#bug-reporting)了解详细说明。

## 快速开始

### 我刚安装了 TabRest。我应该先做什么？

1. **测试手动卸载**：打开几个标签页，点击 TabRest 图标 → "卸载其他标签页"
2. **检查默认设置**：打开设置页面，查看计时器和保护
3. **自定义白名单**：添加重要站点（邮件、文档、聊天应用）
4. **学习快捷键**：设置[键盘快捷键](/docs/keyboard-shortcuts)以实现更快的控制

### 我应该使用白名单还是黑名单模式？

**使用白名单模式**（默认）如果：
- 您想保护少数几个重要站点
- 大多数站点都可以安全卸载

**使用黑名单模式** 如果：
- 您想保护大多数站点
- 您只想激进地卸载少数几个内存占用高的站点（新闻、社交媒体）

参见[白名单配置](/docs/whitelist-config)了解示例。

### 实现最大内存节省的最佳设置是什么？

**激进配置**：
- **计时器**：5 分钟（最短）
- **内存阈值**：60%（激进）
- **启动时卸载**：启用
- **保护固定标签页**：禁用
- **保护音频标签页**：禁用
- **保护表单标签页**：启用（保持开启以避免数据丢失）
- **白名单**：空或最少（仅关键站点）

**警告**：这可能会打断工作流。请根据您的使用情况测试和调整。

### 普通用户的最佳设置是什么？

**平衡配置**（默认）：
- **计时器**：30 分钟
- **内存阈值**：80%
- **启动时卸载**：启用
- **保护固定标签页**：启用
- **保护音频标签页**：启用
- **保护表单标签页**：启用
- **白名单**：YouTube、Gmail、Google Docs

这可在不打断正常浏览的情况下节省内存。
