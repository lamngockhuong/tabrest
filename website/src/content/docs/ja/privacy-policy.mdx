# プライバシーポリシー

**最終更新日**: 2026 年 4 月 29 日

TabRest はあなたのプライバシーの保護に取り組んでいます。このポリシーは、私たちが収集する（またはしない）データと、その使われ方について説明します。

## 概要

- ✅ **デフォルトでは個人データを収集しません**
- ✅ **デフォルトではネットワークリクエストを行いません。すべてのデータはお使いのデバイスに留まります**
- ✅ **トラッキング、分析、行動テレメトリーは一切行いません**
- ✅ **匿名のクラッシュレポートは完全にオプトインです（デフォルトはオフ。[エラーレポート](#error-reporting)を参照）**
- ✅ **オープンソースで監査可能です**

## データ収集

TabRest は動作を 2 つの明確なモードに分けています。デフォルトでは、拡張機能は完全にローカルです。1 つの機能、匿名のクラッシュレポートはオプトインであり、あなたが明示的に有効にした後にのみ、サニタイズされたペイロードを Sentry に送信します。

### 収集するもの

| データの種類 | デフォルト（初期状態） | エラーレポートにオプトインした場合 |
| --- | --- | --- |
| 閲覧履歴（訪問する URL） | ❌ 決して収集しません | ❌ 決して収集しません（スタックトレース内の URL はクライアント側で `[REDACTED]` に伏せられます） |
| ページの内容、タイトル、フォームデータ | ❌ 決して収集しません | ❌ 決して収集しません |
| 個人情報（名前、メール、IP） | ❌ 決して収集しません | ❌ メール/IP はクライアント側で伏せられ、Sentry のペイロードは `user.ip_address: null` を設定します |
| 使用状況の分析 / 行動テレメトリー | ❌ 決して収集しません | ❌ 決して収集しません（ページビュー、クリック、機能利用の追跡なし） |
| クラッシュレポート | ❌ 決して収集しません | ✅ サニタイズされたエラー名、メッセージ、スタックトレース |
| デバイス / 環境 | ❌ 決して収集しません | ✅ 拡張機能のバージョン、ブラウザ + OS の文字列 |
| 集計カウンター | ❌ 決して収集しません | ✅ タブ数（合計/破棄済み/固定/音声あり）、RAM 統計、設定値、インストール日 |
| ホワイトリスト / ブラックリストの内容 | ❌ 決して収集しません | ❌ 除外されます。エントリ数のみ |

オプトインのエラーレポートについては[エラーレポート](#error-reporting)で詳しく説明します。オプション → プライバシーと診断でいつでも無効にできます。

### ローカルに保存するもの

TabRest は次のデータを**お使いのデバイスにのみ**保存します:

| データの種類 | 保存場所 | 目的 | 同期？ |
| ------------ | ---------------------- | ----------------------------------------- | ------- |
| 設定 | `chrome.storage.sync` | ユーザー設定（タイマー、しきい値など） | はい\* |
| ホワイトリスト | `chrome.storage.sync` | 保護対象のドメイン | はい\* |
| タブアクティビティ | `chrome.storage.local` | タブごとの最終アクティビティ時刻（LRU 用） | いいえ |
| 統計 | `chrome.storage.local` | アンロードされたタブ数、節約されたメモリ | いいえ |

\*ブラウザの設定で Chrome sync を有効にしている場合、Chrome Sync 経由で同期されます。私たちはこの同期データを制御したりアクセスしたりしません。

## Chrome の権限

TabRest は次の権限を必要とします:

### `tabs` - 「閲覧履歴の読み取り」

**実際に行うこと**:

- ホワイトリストと照合するためにタブの URL を読み取る
- タブの状態（固定、音声、アクティブ）を読み取る
- `chrome.tabs.discard()` を呼び出してタブをアンロードする

**行わないこと**:

- 閲覧履歴をどこかに送信する
- どのサイトを訪問したかを追跡する
- 閲覧履歴を保存する

**Chrome がこのラベルを表示する理由**: `tabs` 権限はタブの URL へのアクセスを与えます。Chrome の権限ラベルは一般的なもので、実際の使用状況を反映していません。

### `storage` - データをローカルに保存

**行うこと**:

- 設定（タイマー、しきい値、保護）を保存する
- ホワイトリストのエントリを保存する
- LRU 追跡のためにタブアクティビティを保存する
- 統計（アンロードされたタブ数、節約されたメモリ）を追跡する

**すべてのデータはお使いのデバイスに留まります。** 外部ストレージやクラウドサービスはありません。

### `alarms` - 定期タスクをスケジュール

**行うこと**:

- 定期的なタブチェックをスケジュールする（1 分ごと）
- メモリしきい値チェックをスケジュールする（30 秒ごと）

**必要な理由**: Chrome のサービスワーカーはアイドル時に終了します。アラームはサービスワーカーを起動してチェックを実行させます。

### `system.memory` - メモリ使用状況を読み取る

**行うこと**:

- システムの合計 RAM と利用可能な RAM を読み取る
- メモリ使用率を計算する
- しきい値を超えたときにアンロードをトリガーする

**行わないこと**:

- その他のシステム情報にアクセスする
- メモリデータを収集または保存する
- メモリデータをどこかに送信する

### `contextMenus` - 右クリックメニュー項目を追加

**行うこと**:

- 右クリックのコンテキストメニューに「タブをアンロード」オプションを追加する（今後の機能）

### `tabGroups` - タブグループ情報を読み取る

**行うこと**:

- タブグループの名前と色を読み取る
- ポップアップにグループを表示する
- グループ単位でタブをアンロードする

**行わないこと**:

- タブグループを変更または作成する
- グループ情報を共有する

### オプションのホスト権限（v0.0.4 以降）

TabRest は、特定の機能を有効にしたときのみ**オプション**のホスト権限をリクエストします:

**`http://*/*` と `https://*/*`（オプション）**

**リクエストされるタイミング**: 次のいずれかの機能を有効にした場合のみ:
- フォーム保護（「未保存フォームのあるタブを保護」設定）
- 破棄されたタブのタイトル接頭辞（アンロードされたタブのタイトルに記号を付ける）

**行うこと**:
- フォーム保護が有効なときにフォーム検出スクリプトを注入する
- 接頭辞設定が有効なときにタイトル更新スクリプトを注入する

**行わないこと**:
- 機能が有効なときのみ実行し、常時ではありません
- ページの内容を収集または保存する
- 機密データにアクセスする

**制御方法**:
- 設定 → 「未保存フォームのあるタブを保護」または「破棄接頭辞を表示」で有効/無効を切り替えます
- 機能を無効にした場合、Chrome で権限を取り消せます:
  1. `chrome://extensions` を開きます
  2. TabRest → **詳細**を見つけます
  3. **ホストの権限**までスクロールします
  4. 機能が無効な場合は **削除**をクリックします
- 機能を再度有効にすると、Chrome が再度プロンプトを表示します

**オプションである理由**: 本当にホストアクセスが必要な拡張機能だけがそれをリクエストします。TabRest は必要なときのみ求めることで、あなたのプライバシーを尊重します。

## サードパーティサービス

| サードパーティサービス | デフォルト | オプトインした場合 |
| --- | --- | --- |
| 分析（Google Analytics など） | ❌ 決して使用しません | ❌ 決して使用しません |
| 広告ネットワーク | ❌ 決して使用しません | ❌ 決して使用しません |
| コード読み込み用の CDN | ❌ 決して使用しません（拡張機能のコードはバンドルされローカルで実行されます） | ❌ 決して使用しません |
| 外部 API | ❌ なし | ❌ なし |
| エラー追跡（Sentry SaaS、US リージョン） | ❌ 無効。リクエストは送信されません | ✅ オプトイントグルをチェックした後にのみ有効になります。正確なペイロードは[エラーレポート](#error-reporting)を参照してください。 |

**その他のすべてのコードはお使いのブラウザ内でローカルに実行されます。** Sentry のエンドポイント（`*.ingest.us.sentry.io`）は、TabRest が接続しうる唯一のネットワーク宛先であり、エラーレポートを有効にした場合のみです。

## データ共有

TabRest は、サードパーティ企業、広告主、分析プロバイダー、その他の拡張機能とデータを共有**しません**。

**2 つの例外:**

1. **Chrome Sync**（Chrome で有効にした場合）: Chrome 自体が設定とホワイトリストをデバイス間で同期します。私たちはこの同期データを制御したりアクセスしたりしません。
2. **Sentry SaaS**（エラーレポートにオプトインした場合のみ）: サニタイズされたクラッシュペイロードが Sentry の US リージョンの取り込みエンドポイントに送信されます。完全なペイロード仕様、保持期間、オプトアウト手順は[エラーレポート](#error-reporting)を参照してください。

## Chrome Sync

TabRest は設定とホワイトリストに `chrome.storage.sync` を使用します。ブラウザで Chrome Sync が有効な場合:

- **同期される内容**: 設定とホワイトリストのみ
- **制御者**: TabRest ではなく Google Chrome
- **無効にする方法**: Chrome の設定で Chrome Sync をオフにします
- **プライバシー**: 私たちのものではなく [Google のプライバシーポリシー](https://policies.google.com/privacy)に準拠します

**私たちはあなたの Chrome Sync データにアクセスも制御もしません。**

## データセキュリティ

デフォルトでは、TabRest は:

- データをローカルにのみ保存します
- ネットワークリクエストを行いません
- サーバー側のコンポーネントを含みません

**エラーレポートが無効な場合（デフォルト）、漏洩するデータはありません。** あなたのデータは、有効にした場合の Chrome Sync を除いて、決してデバイスの外に出ません。エラーレポートにオプトインした場合、サニタイズされたクラッシュペイロード（URL なし、PII なし）が HTTPS 経由で Sentry の US リージョンの取り込みエンドポイントに送信され、30 日間保持されます。

### コードの完全性

- **オープンソース**: 完全なソースコードは [github.com/lamngockhuong/tabrest](https://github.com/lamngockhuong/tabrest) で入手可能です
- **監査可能**: 誰でもコードをレビューできます
- **難読化なし**: プレーンな JavaScript で、ミニファイやバンドルはありません
- **MIT ライセンス**: 自由に検査、変更、再配布できます

## 子どものプライバシー

TabRest は、13 歳未満の子ども（またはお住まいの地域における同等の最低年齢）を含む、誰からもデータを収集しません。

## データの保持

- **設定とホワイトリスト**: 拡張機能をアンインストールするか Chrome のストレージをクリアするまで保存されます
- **タブアクティビティ**: 拡張機能をアンインストールするまで保存されます
- **統計**: 統計をリセットするか拡張機能をアンインストールするまで保存されます

**すべてのデータを削除するには**:

1. `chrome://extensions` を開きます
2. TabRest → **削除**を見つけます
3. すべてのデータが即座に削除されます

## このポリシーの変更

このポリシーを随時更新する場合があります。変更は次のように行われます:

- このページに掲載されます
- 新しい「最終更新日」で更新されます
- GitHub リポジトリに反映されます

**重要な変更**は次の方法で告知されます:

- GitHub のリリースノート
- 拡張機能の更新説明（Chrome ウェブストア）

## エラーレポート

_Sentry SaaS（US リージョン）経由のクラッシュレポート。_

TabRest は、バグをより早く修正するために匿名のクラッシュレポートを送信できます。**この機能はデフォルトでオフです。** オプション → プライバシーと診断でオプトインする必要があります。

**有効なときに送信されるもの:**

- サニタイズされたエラー名、メッセージ、スタックトレース（URL、メールアドレス、IP アドレスは `[REDACTED]` に置き換えられます）
- 拡張機能のバージョン、ブラウザ名とバージョン、OS プラットフォームの文字列
- タブ数のみ: 合計、破棄済み、固定、音声あり、ウィンドウ数。URL やページタイトルはなし
- システムメモリ統計: 使用率、合計 RAM、利用可能な RAM
- 設定値（自動アンロードタイマー、RAM しきい値、機能フラグなど）。ホワイトリストとブラックリストのドメインは**除外**され、そのエントリ数のみが含まれます
- 累積統計: 全期間および当日のサスペンドされたタブ数、インストール日

**決して送信されないもの:**

- 訪問するページの URL
- ページのタイトル、内容、フォームデータ、スクロール位置
- メールアドレスや IP アドレス（送信前にクライアント側で伏せられます）
- あなたのホワイトリストやブラックリストのドメイン
- レポートをあなたに結びつけられるいかなる識別子

**手動バグ報告:** ポップアップのバグ報告モーダルで「Sentry に送信」をクリックすると、サニタイズされた説明（PII を除去）と上記の診断スナップショットが含まれます。それ以上は含まれません。

**サードパーティ:** レポートは [Sentry](https://sentry.io)（US リージョン、`*.ingest.us.sentry.io`）に送信されます。彼らのデータ取り扱いについては [Sentry のプライバシーポリシー](https://sentry.io/privacy/)を参照してください。

**あなたの IP アドレス:** Sentry のペイロードは明示的に `user.ip_address: null` を設定するため、Sentry はあなたの IP をエラーイベントに関連付けません。

**データの保持:** Sentry はデフォルトでイベントを 30 日間保持し、その後自動的に削除します。

**無効にする方法:** 拡張機能のオプションページ → プライバシーと診断 → 「匿名エラーレポートを有効にする」のチェックを外します。すでに送信されたレポートは保持期限の満了時に削除され、取り消すことはできません。

**カスタム DSN:** 上級ユーザーはオプションで独自の Sentry DSN を指定できます。宛先に関わらず、同じサニタイズルールが適用されます。

## 法的事項

### 管轄

この拡張機能は、個人開発者である Lam Ngoc Khuong によって開発および保守されています。

### コンプライアンス

TabRest は次に準拠しています:

- Chrome ウェブストア デベロッパー プログラム ポリシー
- EU 一般データ保護規則（GDPR） - データを一切収集しないことによって
- カリフォルニア州消費者プライバシー法（CCPA） - データを一切収集しないことによって

### あなたの権利

TabRest は個人データを一切収集しないため:

- アクセス、変更、削除するデータはありません
- エクスポートするデータはありません
- 異議を唱えるべきデータ処理活動はありません

すべてのデータはお使いのデバイスにローカルで保存され、あなたの完全な管理下にあります。

## 透明性

TabRest は透明性を持って構築されています:

- **オープンソース**: MIT ライセンス、公開リポジトリ
- **分析や行動テレメトリーなし**: コードベースのどこにもページビュー、クリック、機能利用の追跡コードはありません
- **デフォルトで外部リクエストなし**: レポートを無効にしてネットワークトラフィックを監査すれば、TabRest からのものは何も見つかりません
- **1 つのオプトインのネットワーク宛先**: エラーレポートを有効にした場合のみの Sentry の US 取り込みエンドポイントのみ
- **シンプルなアーキテクチャ**: バニラ JavaScript、最小限の依存関係、監査が容易

**ご自身で確認を**: リポジトリをクローンしてソースコードを検査してください。分析、トラッキング、文書化されたオプトインのエラーレポーター以外のネットワークリクエストは見つかりません。

---

**要するに**: TabRest はプライバシー第一の拡張機能です。デフォルトでは、あなたのデータはお使いのデバイスに留まります。私たちは何も収集せず、何も送信せず、何も追跡しません。1 つの機能であるエラーレポートは完全にオプトインで、明確に文書化されています。
