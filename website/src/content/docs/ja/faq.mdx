# よくある質問

TabRest とタブのアンロードに関するよくある質問です。

## 一般的な質問

### 「アンロード」とはどういう意味ですか？

**アンロード**とは、タブをタブバーに表示したまま、メモリから取り除くことを意味します。タブはタイトルとファビコン付きで表示されますが、クリックするまでページの内容は読み込まれません。

これはタブを**閉じる**こととは異なります。閉じるとタブバーから完全に取り除かれます。

### アンロードはタブを閉じることと同じですか？

いいえ。主な違いは次のとおりです:

| 特徴 | アンロードされたタブ | 閉じたタブ |
|---------|--------------|------------|
| タブバーに表示 | ✅ はい | ❌ いいえ |
| メモリ使用量 | 非常に少ない（約 1 MB） | なし |
| 復元方法 | クリックで再読み込み | 履歴から再度開く |
| 位置を保持 | ✅ はい | ❌ いいえ |
| フォームデータを保持 | ✅ はい | ❌ いいえ |

### TabRest はどれくらいメモリを節約しますか？

メモリの節約量は使用するウェブサイトによって異なります:

- **シンプルなページ**: タブあたり 50〜100 MB
- **複雑なウェブアプリ**: タブあたり 200〜500 MB
- **重いサイト**（動画、デザインツール）: タブあたり 500 MB 以上

**例**: 一般的な 20 個のタブをアンロードすると、2〜4 GB の RAM を解放できます。

### Chrome を閉じているときも TabRest は動作しますか？

いいえ。TabRest は Chrome が開いているときのみ動作します。ただし:

- **起動時アンロード**（デフォルトで有効）は、Chrome の再起動時にタブをアンロードします
- **設定はブラウザセッションをまたいで保持されます**
- **ホワイトリストは他のデバイスと同期されます**（Chrome sync が有効な場合）

### TabRest はオフラインで動作しますか？

はい。TabRest は次の理由で完全にオフラインで動作します:

- すべてのロジックがブラウザ内でローカルに実行されます
- ネットワークリクエストは不要です
- 外部サーバーは関与しません

**注意**: アンロードされたタブの再読み込みには、ページがキャッシュされていない場合、インターネット接続が必要です。

## プライバシーとセキュリティ

### TabRest は私のデータを収集しますか？

**いいえ。** TabRest は個人データを一切収集しません。

- トラッキングや分析なし
- 外部サーバーへのデータ送信なし
- テレメトリーやクラッシュレポートなし

詳細は[プライバシーポリシー](/docs/privacy-policy)を参照してください。

### TabRest はどのようなデータを保存しますか？

TabRest は Chrome 内にローカルで保存します:

| データの種類 | ストレージ | 目的 |
|-----------|---------|---------|
| 設定 | `chrome.storage.sync` | ユーザー設定 |
| ホワイトリスト | `chrome.storage.sync` | 保護対象のドメイン |
| タブアクティビティ | `chrome.storage.local` | LRU 追跡 |
| 統計 | `chrome.storage.local` | 使用統計 |

**すべてのデータはお使いのデバイスに留まり**、Chrome sync 経由で同期されます（有効な場合）。

### TabRest はなぜ「閲覧履歴の読み取り」権限を必要とするのですか？

この権限（`tabs`）は次のために必要です:

- タブのタイトルと URL を読み取る（ホワイトリストのマッチング用）
- タブの状態を検出する（固定、音声、アクティブ）
- `chrome.tabs.discard()` でタブをアンロードする

**TabRest は次のことをしません**:
- 閲覧履歴をどこかに送信する
- どのサイトを訪問したかを追跡する
- 第三者とデータを共有する

Chrome の権限ラベルは一般的なもので、実際の使用状況を反映していません。

### TabRest はオープンソースですか？

はい。TabRest は MIT ライセンスで、[GitHub で入手可能](https://github.com/lamngockhuong/tabrest)です。

次のことができます:
- ソースコードを監査する
- 問題を報告する
- 改善に貢献する
- 自分用にフォークする

## 技術的な質問

### Chrome Manifest V3 とは何ですか？

Manifest V3（MV3）は Chrome の最新の拡張機能プラットフォームです。V2 との主な違い:

- バックグラウンドページの代わりに **サービスワーカー**（より効率的）
- **より厳格な権限**（より高いセキュリティ）
- **より優れたパフォーマンス**（Chrome は MV3 拡張機能を優先します）

TabRest は最新の Chrome ブラウザ向けに MV3 で構築されています。

### TabRest はブラウザを遅くしますか？

いいえ。TabRest はパフォーマンスへの影響を最小限に抑えるよう設計されています:

- **サービスワーカー**: 必要なときのみ実行し、アイドル時には終了します
- **定期チェック**: 1 分に 1 回（タブ）と 30 秒ごと（メモリ）
- **効率的な API**: ネイティブな Chrome API を使用し、重い計算は行いません

**証明**: Chrome のタスクマネージャー（`Shift+Esc`）を確認してください。TabRest は通常 10 MB 未満しか使用しません。

### TabRest はどれくらい速く非アクティブなタブをアンロードできますか？

自動アンロードタイマーの最小値は **5 分**です。利用可能なプリセット: 5、10、15、30 分、1、2、4 時間。

**理由**:
- アクティブなワークフローの中断を防ぎます
- バックグラウンドタスクが完了する時間を与えます
- メモリの節約と使いやすさのバランスをとります

**代替案**: 即座にアンロードするには手動アンロードボタンを使用します。

### TabRest はなぜ Chrome の内部ページをアンロードできないのですか？

Chrome のセキュリティモデルにより、拡張機能は次を変更できません:

- `chrome://` ページ（設定、拡張機能など）
- `chrome-extension://` ページ
- Chrome ウェブストアのページ

これは Chrome のコア機能を悪意のある拡張機能から保護します。

### TabRest は他の Chromium ブラウザで動作しますか？

TabRest は Manifest V3 をサポートする Chromium ベースのブラウザで動作するはずです:

- ✅ Google Chrome 88 以降
- ✅ Microsoft Edge 88 以降
- ✅ Brave 88 以降
- ✅ Opera 74 以降
- ❌ Firefox（異なる拡張機能 API）
- ❌ Safari（異なる拡張機能 API）

**注意**: 公式サポートは Google Chrome のみです。他のブラウザは動作する可能性がありますが、テストされていません。

## 機能に関する質問

### サイドパネルモードを有効にするには？

TabRest はポップアップの代わりに常設サイドバーで開くことができます:

1. TabRest アイコン → **⚙️ 設定**ボタンをクリックします
2. **ツールバーアクション**セクションまでスクロールします
3. **サイドパネルで開く**をオンに切り替えます
4. 次回のクリックで TabRest がポップアップの代わりにサイドバーで開きます
5. タブを切り替えてもパネルは開いたままになります

**注意**: Chrome 114 以降が必要です。利用できない場合は、ブラウザが古すぎます。

### 重複タブを閉じるには？

組み込みの重複タブクローザーを使用します:

1. TabRest のポップアップを開きます
2. **その他のアクション**セクションまでスクロールします
3. **重複を閉じる**をクリックします
4. TabRest は現在のウィンドウをスキャンし、最も古いタブを残して新しい重複タブを閉じます

**例**: 誤って `github.com` を 3 回開いた場合、TabRest はタブ #2 と #3 を閉じ、タブ #1 を残します。

### ホワイトリストやセッションをバックアップするには？

設定を JSON にエクスポートして保存/共有します:

1. TabRest アイコン → **⚙️ 設定**をクリックします（またはポップアップ → セッションセクションを開きます）
2. **ホワイトリスト**または**セッション**セクションまでスクロールします
3. **エクスポート**をクリックします
4. JSON がクリップボードにコピーされます
5. ファイルに保存するかメールで送信します

別のデバイスで復元するには:
1. **インポート**をクリックします
2. クリップボードから JSON を貼り付けます
3. 新しい項目が既存のリストに追加されます（重複は自動的に削除されます）

### サスペンド警告トーストとは何ですか？

TabRest がタブを自動アンロードする前に表示される、3 秒間のページ上の通知です:

- **表示内容**: ページ上に「3 秒後にタブをサスペンドします...」というメッセージ
- **理由**: 作業を保存したり、アンロードをキャンセルしたりする機会を与えます
- **キャンセル方法**:
  - タブをクリックする（フォーカスを切り替える）
  - ページで音声を再生する
  - フォームフィールドを編集する
  - タブをスヌーズする
- **カスタマイズ可能**: 設定で遅延を調整できます（1〜10 秒）

**例**: ブラウザから離れていて、あるタブがメモリのためにアンロードされそうになります。警告が表示され、まだ使用中だった場合にキャンセルする 3 秒間が与えられます。

### TabRest によるタブのアンロードを一時的に止めるには？

**グローバル一時停止**を使用します:

1. TabRest のポップアップ（またはサイドパネル）を開きます
2. 上部付近の一時停止バーを見つけます
3. 期間を選択します: **30 分**、**1 時間**、**2 時間**、または**再開するまで**
4. TabRest は一時停止が終わるまで、すべての自動アンロード（非アクティブタイマー、メモリしきい値、タブごとのメモリしきい値、起動時の自動破棄）を停止します
5. **再開**をいつでもクリックして、自動アンロードを早めにオンに戻せます

**一時停止中も機能するもの**: 手動アンロードアクション（現在/他/左/右をアンロード）、右クリックのコンテキストメニュー、キーボードショートカット、ツールバークリックアクションはすべて引き続き機能します。停止されるのは自動アンロードのみです。

**インジケーター**: 一時停止中はツールバーバッジに ❚❚ が表示され、ポップアップのタブリストではタブごとのカウントダウンタイマーが非表示になります。

**スヌーズとの違いは？** スヌーズは特定の 1 つのタブまたはドメインを保護します。グローバル一時停止は、すべての場所の自動アンロードを一度に停止する 1 つのスイッチです。

### TabRest は X 個のタブが開いた後にタブを自動アンロードできますか？

現在はできません。TabRest は次に基づいてトリガーします:
- 最後のアクティビティからの経過時間
- システムのメモリしきい値

**回避策**: 積極的なアンロードのために、低いタイマー（5 分）または低いメモリしきい値（60%）を設定します。

### TabRest は特定のウェブサイトのタブのみをアンロードできますか？

はい、**ブラックリストモード**を使用します:

1. 設定を開きます
2. **ブラックリストモード**を有効にします
3. ドメインをリストに追加します
4. それらのドメインのみがアンロードされ、それ以外はすべて保護されます

詳細は[ホワイトリスト設定](/docs/whitelist-config)を参照してください。

### TabRest はホワイトリストで localhost や IP アドレスをサポートしますか？

はい。ローカル開発サーバーをホワイトリストに登録できます:

```
localhost
127.0.0.1
192.168.1.100
::1 (IPv6 loopback)
```

**ユースケース**: 他のサイトをアンロードしたまま、ローカルの開発環境を自動アンロードから保護します。

### TabRest はアンロードされたすべてのタブを一度に復元できますか？

いいえ。アンロードされた各タブを個別にクリックして再読み込みする必要があります。

**理由**: すべてのタブを一度に再読み込みすると、メモリ節約の目的を損なってしまいます。

**回避策**: Chrome の「すべてのタブをブックマーク」を使ってセッションを保存し、必要に応じてタブを再度開きます。

### TabRest はアンロードをスケジュールできますか（例: 毎日午後 9 時）？

現在はできません。TabRest の自動アンロードは継続的であり、スケジュールベースではありません。

**機能リクエスト**: スケジュールベースのアンロードは将来のバージョンで計画されています。[GitHub Issues](https://github.com/lamngockhuong/tabrest/issues) で追跡してください。

### TabRest は他のウィンドウのタブをアンロードできますか？

はい。TabRest はすべての Chrome ウィンドウにわたって機能します:

- 自動アンロードはすべてのウィンドウをチェックします
- ポップアップからの手動アンロードは現在のウィンドウのみに影響します
- キーボードショートカットは現在のウィンドウのみに影響します

### TabRest はタブサスペンダー拡張機能と併用できますか？

TabRest は他のタブ管理拡張機能と併用できるはずですが:

- **競合の可能性**: 両方の拡張機能がタブのアンロードを競い合う場合があります
- **機能の重複**: 両方は必要ないかもしれません

**推奨**: 一度に 1 つのタブ管理拡張機能を使用してください。

### TabRest は Chrome の組み込みメモリセーバーとどう連携しますか？

Chrome 108 以降には `chrome://settings/performance` に **メモリセーバー**が搭載されており、ブラウザレベルで非アクティブなタブを破棄します。TabRest とメモリセーバーは独立して動作します:

- **メモリセーバーは TabRest に問い合わせません**。タブをメモリセーバーの対象外にする Chrome 拡張機能 API は存在しません。
- **TabRest のホワイトリストとスヌーズは TabRest 自身のアンロードパイプラインにのみ適用されます**。それらは Chrome が同じタブを破棄するのを防ぎません。

**両方が有効な場合**、ホワイトリストに登録したタブでも Chrome によって破棄される可能性があります。これを防ぐには、次のいずれかを行います:

1. Chrome メモリセーバーをオフにして、TabRest にすべてを管理させる（きめ細かな制御を望むユーザーに推奨）、または
2. 両方をオンのままにし、重要なドメインを `chrome://settings/performance` の **常にこれらのサイトをアクティブにする**リストにも追加する

詳細は[ホワイトリスト設定](/docs/whitelist-config#coexistence-with-chrome-memory-saver)を参照してください。

### ホワイトリストに登録したタブがまだアンロードされるのはなぜですか？

最もよくある原因は、Chrome メモリセーバーが TabRest と並行して動作していることです。TabRest のホワイトリストは Chrome のメモリセーバーに影響しません。修正方法は上記の質問を参照してください。

その他に考えられる原因:

- ドメイン形式が間違っている（`https://example.com/path` ではなく `example.com` を使用）
- ブラックリストモードが有効になっている（ホワイトリストがアンロードリストになる）
- 別のタブ管理拡張機能もインストールされている

完全な診断手順は[トラブルシューティング → ホワイトリストが機能しない](/docs/troubleshooting#whitelist-not-working)を参照してください。

## トラブルシューティング

### アンロード後にすぐ再読み込みされるタブがあるのはなぜですか？

**考えられる原因**:

1. **ウェブサイトの自動再読み込み**: 一部のサイトは JavaScript を使ってアンロードを検出し、強制的に再読み込みします
2. **拡張機能の競合**: 別の拡張機能がタブを再読み込みしている可能性があります
3. **Chrome sync**: タブの同期が別のデバイスからタブを復元する場合があります

**解決策**: これらのサイトをホワイトリストに追加してアンロードを防ぎます。

### バッジのカウントが間違った数字を表示するのはなぜですか？

**考えられる原因**:

1. **バッジが無効**: 設定 → 「バッジカウントを表示」が有効か確認します
2. **サービスワーカーが終了した**: バッジはサービスワーカーが実行されているときに更新されます
3. **拡張機能のバグ**: 拡張機能を再読み込みします

**解決策**: ポップアップを開くと（サービスワーカーが起動します）、バッジが更新されるはずです。

### 設定をエクスポートできますか？

UI から直接はできません。設定は `chrome.storage.sync` に保存され、Chrome sync 経由で自動的に同期されます。

**手動エクスポート**（上級者向け）:
1. `chrome://extensions` を開きます
2. TabRest → **詳細** → **ビューを検証: service worker**
3. コンソールで:
   ```javascript
   chrome.storage.sync.get('settings', console.log);
   ```
4. 出力をコピーします

### バグを報告するには？

詳しい手順は[トラブルシューティング → バグ報告](/docs/troubleshooting#bug-reporting)を参照してください。

## はじめに

### TabRest をインストールしたばかりです。最初に何をすべきですか？

1. **手動アンロードをテスト**: いくつかのタブを開き、TabRest アイコン → 「他のタブをアンロード」をクリックします
2. **デフォルト設定を確認**: 設定ページを開き、タイマーと保護を確認します
3. **ホワイトリストをカスタマイズ**: 重要なサイト（メール、ドキュメント、チャットアプリ）を追加します
4. **ショートカットを学ぶ**: より素早い操作のために[キーボードショートカット](/docs/keyboard-shortcuts)を設定します

### ホワイトリストとブラックリストのどちらのモードを使うべきですか？

次の場合は**ホワイトリストモード**（デフォルト）を使用します:
- 重要なサイトをいくつか保護したい
- ほとんどのサイトは安全にアンロードできる

次の場合は**ブラックリストモード**を使用します:
- ほとんどのサイトを保護したい
- メモリを多く消費する少数のサイト（ニュース、ソーシャルメディア）のみを積極的にアンロードしたい

例については[ホワイトリスト設定](/docs/whitelist-config)を参照してください。

### メモリ節約を最大化するための最適な設定は？

**積極的な設定**:
- **タイマー**: 5 分（最小）
- **メモリしきい値**: 60%（積極的）
- **起動時アンロード**: 有効
- **固定タブを保護**: 無効
- **音声タブを保護**: 無効
- **フォームタブを保護**: 有効（データ損失を避けるためオンのまま）
- **ホワイトリスト**: 空または最小限（重要なサイトのみ）

**警告**: これはワークフローを中断する可能性があります。使用状況に応じてテストと調整を行ってください。

### 一般的なユーザーにとって最適な設定は？

**バランスの取れた設定**（デフォルト）:
- **タイマー**: 30 分
- **メモリしきい値**: 80%
- **起動時アンロード**: 有効
- **固定タブを保護**: 有効
- **音声タブを保護**: 有効
- **フォームタブを保護**: 有効
- **ホワイトリスト**: YouTube、Gmail、Google ドキュメント

これは通常のブラウジングを中断することなくメモリを節約します。
