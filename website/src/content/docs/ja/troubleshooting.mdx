# トラブルシューティング

TabRest を使用する際のよくある問題の解決策です。

## タブがアンロードされない

**問題**: 「アンロード」をクリックしても何も起こらない、またはタブがアンロードを拒否します。

### 原因 1: タブが保護されている

TabRest はデフォルトで特定のタブを保護します。タブが次のいずれかに該当しないか確認してください:

- **固定**: 固定タブは保護されます（「固定タブをアンロード」が有効でない限り）
- **音声を再生中**: アクティブな音声のあるタブは保護されます（「音声タブを保護」が無効でない限り）
- **未保存フォームあり**: 変更された入力フィールドのあるタブは保護されます（「フォームタブを保護」が無効でない限り）
- **ホワイトリスト登録済み**: ドメインがホワイトリストに載っています

**解決策**: 設定で該当する保護を無効にするか、ホワイトリストからドメインを削除します。

### 原因 2: タブがアクティブ

Chrome は現在アクティブな（フォーカスされている）タブをアンロードできません。

**解決策**: 先に別のタブに切り替えてから、対象のタブをアンロードします。

### 原因 3: Chrome の内部ページ

Chrome は拡張機能が内部ページをアンロードするのを防ぎます:

- `chrome://` ページ（設定、拡張機能など）
- `chrome-extension://` ページ
- Chrome ウェブストアのページ
- 新しいタブページ

**解決策**: これらのページは仕様上アンロードできません。使用するメモリはごくわずかです。

### 原因 4: タブを読み込んだばかり

Chrome は、開いたばかりまたは再読み込みしたばかり（5 秒未満）のタブのアンロードを防ぐことがあります。

**解決策**: 数秒待ってから再試行します。

### 原因 5: 拡張機能の権限

TabRest がタブをアンロードするには `tabs` 権限が必要です。

**解決策**:
1. `chrome://extensions` を開きます
2. TabRest → **詳細**を見つけます
3. 権限に「閲覧履歴の読み取り」が含まれていることを確認します
4. 見当たらない場合は、拡張機能を削除して再インストールします

## 拡張機能が動作しない

**問題**: TabRest アイコンが反応しない、または自動アンロードがトリガーされません。

### 拡張機能のステータスを確認

1. `chrome://extensions` を開きます
2. リストで TabRest を見つけます
3. トグルが **オン**（青）であることを確認します
4. エラー表示（赤いテキスト）がないか確認します

### 拡張機能を再読み込み

1. `chrome://extensions` を開きます
2. TabRest を見つけます
3. **更新アイコン**（↻）をクリックします
4. 機能を再度テストします

### サービスワーカーを確認

1. `chrome://extensions` を開きます
2. TabRest → **詳細**を見つけます
3. **ビューを検証**までスクロールします
4. **service worker** をクリックします
5. コンソールでエラーを確認します

**よくあるエラー**:
- `Storage quota exceeded`: 拡張機能のストレージをクリアします
- `Alarm not registered`: 拡張機能を再読み込みします
- `Permission denied`: 拡張機能を再インストールします

### 拡張機能を再インストール

再読み込みで解決しない場合:

1. **設定をエクスポート**（任意）: ホワイトリストと設定をメモします
2. `chrome://extensions` を開きます
3. TabRest → **削除**を見つけます
4. Chrome ウェブストアまたはソースから再インストールします
5. 設定を復元します

## キーボードショートカットが動作しない

**問題**: ショートカットを押しても何も起こりません。

### ショートカットが設定されているか確認

1. `chrome://extensions/shortcuts` を開きます
2. TabRest のセクションを見つけます
3. ショートカットが割り当てられている（空でない）ことを確認します

**解決策**: 空の場合は、手動でショートカットを設定します。

### 競合を確認

ショートカットは次のものと競合する場合があります:
- 他の拡張機能
- Chrome ブラウザのショートカット
- オペレーティングシステムのショートカット

**解決策**:
1. `chrome://extensions/shortcuts` を開きます
2. ショートカットに「Chrome が使用中」と表示されている場合は、別の組み合わせを選びます
3. 競合する拡張機能のショートカットを無効にします

### 別のコンテキストでテスト

一部のショートカットは次では動作しません:
- Chrome の内部ページ（`chrome://`）
- 拡張機能のページ（`chrome-extension://`）
- Chrome ウェブストア

**解決策**: 通常のウェブページ（例: `example.com`）でテストします。

### 拡張機能を再読み込み

1. `chrome://extensions` を開きます
2. TabRest → 更新アイコンを見つけます
3. ショートカットを再度テストします

## メモリ使用量が高い

**問題**: TabRest を使っても Chrome のメモリ使用量が依然として高いです。

### タブが実際にアンロードされているか確認

1. タスクマネージャーを開きます: `Shift+Esc`（Windows/Linux）または Chrome メニュー → その他のツール → タスクマネージャー
2. 「アンロード済み」とマークされたタブ、または非常に低いメモリ（1 MB 未満）のタブを探します
3. タブが高メモリを示している場合、アンロードされていません

**解決策**: タブ保護の設定（固定、音声、フォーム、ホワイトリスト）を確認します。

### 自動アンロードの設定を確認

1. TabRest の設定を開きます
2. **タイマー**が有効であることを確認します（「なし」に設定されていない）
3. **メモリしきい値**が妥当であることを確認します（60〜95%）
4. **起動時アンロード**を有効にして、ブラウザ再起動時にメモリを解放します

### 拡張機能とバックグラウンドページ

TabRest は次をアンロードできません:
- 拡張機能のページ自体
- 他の拡張機能のサービスワーカー
- Chrome 自身のプロセス

**解決策**: Chrome のタスクマネージャー（`Shift+Esc`）を使って、メモリを多く消費する拡張機能を特定し、無効にします。

### タブが多すぎる

数百のタブを開いている場合:
- 自動アンロードはすべてのタブの処理に時間がかかります
- 一部のタブは保護されているか、最近アクティブだった可能性があります
- 不要になったタブを閉じることを検討してください

**解決策**: 「他のタブをアンロード」を使って、すべての非アクティブなタブを手動で即座にアンロードします。

## 統計が更新されない

**問題**: 統計ページにアンロードされたタブが 0 と表示される、または古い数値が表示されます。

### 統計トグルを確認

1. 設定を開きます
2. **統計を有効にする**がオンであることを確認します
3. オフだった場合はオンにします（統計はその時点から追跡を開始します）

### アンロードが行われているか確認

1. ポップアップを使ってタブを手動でアンロードします
2. 設定 → 統計セクションを開きます
3. カウントが増えたか確認します

**解決策**: それでも統計が更新されない場合は、拡張機能を再読み込みします。

### 統計をリセット

統計が破損している場合:

1. 設定を開きます
2. **統計をリセット**ボタンをクリックします
3. 操作を確認します
4. 統計はゼロから追跡を開始します

## ホワイトリストが機能しない

**問題**: ホワイトリストに登録したサイトが依然としてアンロードされます。

### ドメイン形式を確認

ホワイトリストのエントリはドメイン名のみにする必要があります:

```
✅ Correct: github.com
✅ Correct: mail.google.com
❌ Wrong: https://github.com
❌ Wrong: github.com/user/repo
```

**解決策**: エントリからプロトコル（`https://`）とパス（`/user/repo`）を削除します。

### ブラックリストモードを確認

**ブラックリストモード**が有効な場合、動作が逆になります:
- ホワイトリストがブラックリストになります（アンロードするサイト）
- それ以外のすべてのサイトが保護されます

**解決策**: 通常のホワイトリストの動作を望む場合は、設定でブラックリストモードを無効にします。

### 大文字小文字の区別

ドメインのマッチングは大文字小文字を区別しません:
- `GitHub.com` = `github.com` ✅

**解決策**: 一貫性のために小文字を使用します。

### Chrome メモリセーバーも有効になっている

**Chrome 108 以降でホワイトリストが壊れているように見える場合の最もよくある原因です。**

Chrome には独自のタブ破棄機能（`chrome://settings/performance` → メモリセーバー）が搭載されています。これはブラウザレベルで動作し、拡張機能の状態を無視します。メモリセーバーが有効な場合、ドメインが TabRest のホワイトリストに載っていてもタブを破棄することがあります。メモリセーバーは TabRest に問い合わせないためです。

**これが原因であることを確認する方法**:

1. TabRest のサスペンド警告トーストが最初に表示されずにタブがアンロードされる
2. TabRest が一時停止または無効になっていてもタブがアンロードされる
3. `chrome://settings/performance` でメモリセーバーがオンになっているのが見える

**解決策（いずれか 1 つを選択）**:

- **オプション A**: `chrome://settings/performance` を開いてメモリセーバーをオフにします。これで TabRest が完全に制御します。
- **オプション B**: メモリセーバーをオンのままにしつつ、`chrome://settings/performance` の **常にこれらのサイトをアクティブにする**リストにもドメインを追加します。TabRest のホワイトリストと Chrome のリストを手動で同期させる必要があります。

完全な説明は[ホワイトリスト設定 → Chrome メモリセーバーとの共存](/docs/whitelist-config#coexistence-with-chrome-memory-saver)を参照してください。

## フォーム保護が機能しない

**問題**: 未保存フォームのあるタブがアンロードされます。

### フォーム保護の設定を確認

1. 設定を開きます
2. **未保存フォームのあるタブを保護**が有効であることを確認します

### フォーム検出を確認

TabRest は次のフォームを検出します:
- 変更された `<input>` フィールド（テキスト、メール、パスワードなど）
- 変更された `<textarea>` フィールド
- 変更された `<select>` ドロップダウン

**検出しないもの**:
- リッチテキストエディタ（例: TinyMCE、CKEditor） - 代わりにホワイトリストを使用してください
- キャンバスベースのエディタ
- カスタムの JavaScript フォームライブラリ

**解決策**: 複雑なフォームのあるサイトをホワイトリストに追加します。

### コンテンツスクリプトのタイミング

ページ読み込み直後に非常に素早くフォームを入力した場合、TabRest がまだ検出していない可能性があります。

**解決策**: フォームを入力してから数秒待ってタブを切り替えます。

## サイドパネルの問題

### サイドパネルが開かない

**問題**: ツールバーアイコンをクリックするとサイドパネルではなくポップアップが開く、またはサイドパネルのトグルが利用できません。

**考えられる原因**:

1. **ブラウザが古すぎる**: サイドパネル API には Chrome 114 以降が必要です
2. **設定が有効になっていない**: 設定で「サイドパネルで開く」トグルがオンか確認します

**解決策**:
- Chrome をバージョン 114 以降に更新します
- 設定を開いてサイドパネルのトグルを有効にします
- 拡張機能を再読み込みします（`chrome://extensions` → 更新）

### サイドパネルが閉じる、または消える

**問題**: タブを切り替えるとサイドパネルが閉じる、または予期せず消えます。

**考えられる原因**:

1. **パネルが折りたたまれている**: パネルが端で折りたたまれていないか確認します
2. **サービスワーカーのクラッシュ**: 拡張機能を再読み込みします

**解決策**:
1. パネルが端にある場合は、クリックして展開します
2. 拡張機能を再読み込みします: `chrome://extensions` → TabRest → 更新
3. 「サイドパネルで開く」をオフにして再度オンにします

## フォーム保護の問題

### フォーム保護が権限を要求する

**問題**: 「未保存フォームのあるタブを保護」を有効にすると、ホスト権限を求められます。

**説明**: フォーム保護には、ウェブページに検出スクリプトを注入する必要があります。これにはオプションのホスト権限が必要です。

**解決策**:
1. Chrome が求めたときに **許可**をクリックします
2. 誤って **ブロック**をクリックした場合は、次のように復元できます:
   - `chrome://extensions` を開きます
   - TabRest → **詳細**を見つけます
   - **ホストの権限**までスクロールします
   - **許可**をクリックするか「すべてのサイト」に変更します

### サスペンド警告が表示されない

**問題**: 自動サスペンドの前に警告トーストが表示されません。

**考えられる原因**:

1. **ホスト権限が付与されていない**: サスペンド警告の注入にはホストアクセスが必要です
2. **設定が無効**: 「サスペンド警告を表示」が有効か確認します
3. **サービスワーカーがクラッシュした**: 拡張機能を再読み込みします

**解決策**:
1. 設定を開いて「サスペンド警告を表示」を有効にします
2. ホスト権限を確認します（上記の「フォーム保護が権限を要求する」を参照）
3. 拡張機能を再読み込みします: `chrome://extensions` → TabRest → 更新

## バグ報告

ここに記載されていないバグに遭遇した場合:

### 情報を収集

1. **Chrome のバージョン**: `chrome://version`
2. **TabRest のバージョン**: `chrome://extensions` → TabRest → 詳細
3. **再現手順**: どの操作でバグが発生するか
4. **期待される動作**: 何が起こるべきか
5. **実際の動作**: 実際に何が起こるか
6. **コンソールエラー**: サービスワーカーのインスペクターを開き、エラーを確認します

### GitHub で報告

1. [TabRest Issues](https://github.com/lamngockhuong/tabrest/issues)を開きます
2. 重複を避けるため既存の課題を検索します
3. **New Issue** をクリックします
4. 上記のすべての情報を含めます
5. 役立つ場合はスクリーンショットを追加します

### 緊急の回避策

TabRest が重大な問題を引き起こす場合:

1. 拡張機能を無効にします: `chrome://extensions` → トグルをオフ
2. GitHub でバグを報告します
3. 修正を待つか、ソースから再インストールします
