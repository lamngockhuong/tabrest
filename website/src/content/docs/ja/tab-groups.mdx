# タブグループ

TabRest は Chrome のネイティブなタブグループ機能と連携し、グループ全体を一度にアンロードできます。

## タブグループとは

タブグループは Chrome の機能（Chrome 88 で追加）で、タブをラベル付き・色分けされたグループに整理できます。

**タブグループを作成する**:
1. タブを右クリック → **新しいグループに追加**
2. 名前と色を選びます
3. タブをグループにドラッグします

**Chrome のタブグループ**:
- 折りたたんで省スペース化できます
- 視覚的な整理のために色分けされます
- 一緒に移動できます
- デバイス間で同期されます（Chrome sync が有効な場合）

## タブグループのアンロード

TabRest はポップアップにグループ単位のアンロードを追加します:

### ポップアップから

1. ツールバーの **TabRest アイコン**をクリックします
2. **タブグループ**セクションまでスクロールします
3. グループ名をクリックします
4. そのグループ内のすべてのタブがアンロードされます

**視覚的なインジケーター**:
- グループ名と色がポップアップに表示されます
- 各グループ内のタブ数が表示されます
- アンロードされたタブは 💤 アイコンでマークされます

### グループ内の保護されたタブ

グループをアンロードする場合でも、個々のタブの保護は引き続き適用されます:

| タブの種類 | 動作 |
|----------|----------|
| **固定タブ** | 保護される（「固定タブをアンロード」が無効の場合） |
| **音声タブ** | 保護される（「音声タブを保護」が有効の場合） |
| **フォームタブ** | 保護される（「フォームタブを保護」が有効の場合） |
| **ホワイトリスト登録済み** | 保護される（常に） |

**例**: 10 個のタブがあるグループをアンロードしても、そのうち 2 つに未保存のフォームがある場合、アンロードされるのは 8 つだけです。

## 自動アンロード

TabRest の自動アンロード（タイマーベースおよびメモリしきい値）は、グループ化されたタブを含むすべてのタブにわたって機能します。

**特別なグループ保護はなし**: タブグループは自動アンロードを防ぎません。グループを保護したい場合は:

1. グループのドメインをホワイトリストに追加します
2. グループ内のタブを固定します
3. タブをアクティブに保ちます（定期的に操作します）

## タブグループの設定

TabRest にはタブグループ機能のトグルが含まれています:

**タブグループを有効にする**（デフォルト: 有効）:
- 有効の場合: タブグループセクションがポップアップに表示されます
- 無効の場合: グループがポップアップから非表示になりますが、自動アンロードは引き続き機能します

**無効にするには**:
1. 設定ページを開きます
2. **タブグループを有効にする**をオフに切り替えます
3. 変更は自動的に保存されます

## ユースケース

### プロジェクトごとに整理

さまざまなプロジェクトやコンテキストごとにグループを作成します:

- **仕事**: 会社のメール、プロジェクトドキュメント、課題トラッカー
- **プライベート**: ソーシャルメディア、銀行、ショッピング
- **リサーチ**: 記事、ドキュメント、チュートリアル

コンテキストを切り替える際に、プロジェクト全体をアンロードします。

### セッション管理

グループを一時的なセッションとして使用します:

1. タスクのためにタブを開きます
2. それらをまとめてグループ化します
3. 完了したら、グループ全体をアンロードします
4. 後のためにタブをグループ化したまま保持します

ブックマークより優れている点:
- タブは開いたまま（アンロードされているだけ）
- 復元が速い
- スクロール位置とフォームデータを保持

### メモリ管理

メモリを多く消費するグループを特定します:

1. メモリ集約型のサイト（例: 動画、デザインツール）のグループを作成します
2. 積極的に使用していないときにグループをアンロードします
3. 必要なときに再読み込みします

## ヒント

### グループの命名

素早く識別できるよう、わかりやすい名前を使用します:

```
✅ Good: "Project X Documentation"
✅ Good: "Q1 Planning"
❌ Poor: "Group 1"
❌ Poor: "Tabs"
```

### 色分け

優先度やカテゴリーで色を割り当てます:

- 🔴 赤: 緊急/アクティブな作業
- 🟢 緑: 参照/バックグラウンド
- 🔵 青: プライベートなタスク
- 🟡 黄: 待機中/レビュー
- 🟣 紫: リサーチ/学習

### ホワイトリストとの組み合わせ

重要なグループには、ドメインをホワイトリストに追加します:

1. グループ内のタブを特定します
2. ドメインをホワイトリストに追加します
3. グループは自動的に読み込まれたままになります

## 制限事項

### Chrome 固有の機能

タブグループは Chrome の機能です。TabRest には次が必要です:

- タブグループのサポートには **Chrome 88 以降**
- API でのタブグループの色には **Chrome 89 以降**

**タブグループのないブラウザ**: TabRest のグループ機能は自動的に非表示になります。

### グループレベルの保護なし

TabRest には次ができません:
- グループ全体を自動アンロードから保護する（代わりに個々のタブを保護してください）
- グループを作成または管理する（Chrome の組み込み機能を使用してください）
- ブラウザ間でグループを同期する（Chrome sync が処理します）

### 拡張機能 API の制限

TabRest は `chrome.tabGroups` API を使用しますが、これは:
- グループのプロパティ（名前、色）を変更できません
- グループを作成または削除できません
- グループ情報の読み取りとタブのアンロードのみが可能です

## キーボードショートカット

TabRest はグループ操作用のキーボードショートカットを提供しません。Chrome の組み込みショートカットを使用してください:

| アクション | Windows/Linux | macOS |
|--------|---------------|-------|
| グループを作成 | `Ctrl+G` | `Cmd+G` |
| 次のグループ | `Ctrl+Shift+9` | `Cmd+Shift+9` |
| 前のグループ | `Ctrl+Shift+0` | `Cmd+Shift+0` |

**注意**: これらのショートカットは Chrome のバージョンによって異なる場合があります。

## トラブルシューティング

### グループがポップアップに表示されない

**問題**: タブグループセクションが空です。

**解決策**:
1. **グループを作成**: TabRest は存在するグループのみを表示します
2. **設定を確認**: 「タブグループを有効にする」が有効であることを確認します
3. **拡張機能を再読み込み**: `chrome://extensions` → TabRest → 更新

### グループがアンロードされない

**問題**: グループ名をクリックしても何も起こりません。

**解決策**:
1. **タブ保護を確認**: 一部のタブは保護されている可能性があります（固定、音声、ホワイトリスト）
2. **アクティブなタブを確認**: 現在アクティブなタブはアンロードできません
3. **Chrome のバージョンを確認**: Chrome 88 以降が必要です

### アンロード後にグループが消えた

**問題**: アンロード後にグループが折りたたまれたり非表示になったりします。

**説明**: これは Chrome の動作です。折りたたまれたグループ内のアンロードされたタブは折りたたまれたままになります。

**解決策**: グループのラベルをクリックして展開し、アンロードされたタブを表示します。
