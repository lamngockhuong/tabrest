# ホワイトリスト設定

TabRest では、ホワイトリスト（または逆モードのブラックリスト）を使用して、特定のウェブサイトを自動アンロードから保護できます。

## デフォルトのホワイトリスト

TabRest には次のサイトが初期設定されています:

- `youtube.com` - 動画とプレイリストを保護
- `gmail.com` - メールタブを読み込んだまま維持
- `docs.google.com` - Google ドキュメント、スプレッドシート、スライドを保護

これらは設定で削除やカスタマイズが可能です。

## ホワイトリストへのサイトの追加

### オプションページから

1. TabRest アイコン → **⚙️ 設定**ボタンをクリックします
2. **ホワイトリスト**セクションまでスクロールします
3. ドメイン名を入力します（例: `github.com`）
4. **Enter** を押すか **追加**をクリックします
5. 変更は自動的に保存されます

### ポップアップからのクイック追加

現在のサイトをポップアップから直接クイック追加します:

1. 保護したいサイトに移動します
2. ツールバーの TabRest アイコンをクリックします
3. **現在のサイトをホワイトリストに追加**リンクを探します
4. クリックして現在のドメインを追加します

**注意**: この機能はドメインのみ（例: `example.com`）を追加し、完全な URL は追加しません。

## ドメインマッチングのルール

TabRest はシンプルなドメインマッチングを使用します:

### 完全一致

```
Whitelist: github.com
✅ Matches: https://github.com/user/repo
✅ Matches: http://github.com/explore
❌ No match: https://gist.github.com
❌ No match: https://api.github.com
```

### サブドメインのマッチング

すべてのサブドメインを保護するには、ベースドメインを追加します:

```
Whitelist: google.com
✅ Matches: docs.google.com
✅ Matches: mail.google.com
✅ Matches: drive.google.com
```

**または**特定のサブドメインを追加します:

```
Whitelist: docs.google.com
✅ Matches: https://docs.google.com/document/...
❌ No match: https://mail.google.com
```

### よくあるパターン

| ホワイトリストのエントリ | 保護対象 |
|-----------------|----------|
| `example.com` | `example.com` とすべてのサブドメイン |
| `www.example.com` | `www.example.com` のみ |
| `app.example.com` | `app.example.com` のみ |

## サイトの削除

1. **設定**ページを開きます
2. ホワイトリスト内のドメインを見つけます
3. その隣の **X** ボタンをクリックします
4. 変更は自動的に保存されます

## ブラックリストモード

ホワイトリストの動作を逆にします。リストに載っているサイトを保護する代わりに、リストに載っているサイト**のみ**をアンロードします。

### 有効にする方法

1. **設定**ページを開きます
2. **ブラックリストモード**スイッチを切り替えます
3. リストに載っているサイト以外はすべて保護されます

### ユースケース

**ホワイトリストモード（デフォルト）**: 重要なサイトをいくつか保護し、それ以外をすべてアンロードします

```
Whitelist: gmail.com, docs.google.com
→ Only these sites are protected
→ All other sites can be unloaded
```

**ブラックリストモード**: ほとんどのサイトを保護し、特定のサイトを積極的にアンロードします

```
Blacklist: news.com, social.com
→ Only these sites can be unloaded
→ All other sites are protected
```

**ブラックリストモードを使うタイミング**:
- ほとんどのサイトをデフォルトで保護したい
- メモリを多く消費する少数のサイト（例: ニュースサイト、ソーシャルメディア）のみをアンロードしたい
- アンロードするサイトのリストが短い

## ホワイトリスト + その他の保護

ホワイトリストは他のタブ保護と併用できます:

| 保護の種類 | 優先度 | 説明 |
|-----------------|----------|-------------|
| ホワイトリスト | 最高 | ドメインベースの保護 |
| 固定タブ | 高 | 固定タブを保護（設定可能） |
| 音声タブ | 高 | メディアを再生しているタブを保護（設定可能） |
| フォームタブ | 高 | 未保存フォームのあるタブを保護（設定可能） |

**例**: サイトがホワイトリストに載っていなくても、次の場合はアンロードされません:
- タブが固定されている（かつ「固定タブをアンロード」が無効）
- タブが音声を再生している（かつ「音声タブを保護」が有効）
- タブに未保存のフォームデータがある（かつ「フォームタブを保護」が有効）

## インポート/エクスポート

TabRest は、JSON によるホワイトリスト、ブラックリスト、セッションの簡単なバックアップと復元をサポートしています:

### ホワイトリスト/セッションのエクスポート

1. TabRest アイコン → **⚙️ 設定**をクリックします（またはポップアップ → セッションセクション）
2. **ホワイトリスト**または**セッション**セクションまでスクロールします
3. **エクスポート**ボタンをクリックします
4. JSON がクリップボードにコピーされます
5. テキストファイルに貼り付けて保存します

**エクスポート形式**（v1 スキーマ）:
```json
{
  "version": 1,
  "whitelist": ["github.com", "localhost"],
  "blacklist": [],
  "sessions": [...]
}
```

### ホワイトリスト/セッションのインポート

1. 保存した JSON をクリップボードにコピーします
2. TabRest アイコン → **⚙️ 設定**をクリックします（またはポップアップ → セッションセクション）
3. **インポート**ボタンをクリックします
4. クリップボードから JSON を貼り付けます
5. **確認**をクリックします
6. 新しい項目が既存のリストに追加されます
7. 重複は自動的に削除されます

**インポートの動作**:
- **追加型**: 新しいエントリが既存のホワイトリスト/ブラックリストに追加されます
- **重複排除**: `["github.com"]` をインポートしてすでに存在する場合、重複しません
- **非破壊的**: 既存のエントリを削除しません

### 自動同期

ホワイトリストは `chrome.storage.sync` にも保存され、Chrome Sync を有効にしていれば Chrome ブラウザ間で自動的に同期されます。

**自動同期を無効にするには**:
1. Chrome の設定 → **同期と Google サービス**を開きます
2. **すべてを同期**をオフにするか、**拡張機能**のチェックを外します
3. ホワイトリストはこのデバイスのみに残ります

### 手動バックアップ（上級者向け）

コンソールアクセスを好む技術的なユーザー向け:

1. `chrome://extensions` を開きます
2. TabRest → **詳細**を見つけます
3. **ビューを検証: service worker** までスクロールします
4. コンソールで次を実行します:
   ```javascript
   chrome.storage.sync.get('settings', (data) => {
     console.log(JSON.stringify(data.settings.whitelist));
   });
   ```
5. 出力された JSON 配列をコピーします

## ヒント

### 大きなホワイトリストの管理

- **具体的に**: 積極的に読み込んだまま維持したいサイトのみを追加します
- **定期的に見直す**: 使わなくなったサイトを削除します
- **ブラックリストモードを検討**: ホワイトリストが大きくなったら、ブラックリストモードに切り替えます

### よくあるホワイトリストの例

**開発ワークフロー**:
```
localhost
127.0.0.1
github.com
stackoverflow.com
```

**生産性ツール**:
```
mail.google.com
calendar.google.com
slack.com
notion.so
```

**コミュニケーションアプリ**:
```
discord.com
zoom.us
teams.microsoft.com
```

### ワイルドカードのサポート

TabRest は現在、ワイルドカード（例: `*.google.com`）を**サポートしていません**。各ドメインを個別に追加してください:

```
✅ Add: mail.google.com, drive.google.com, docs.google.com
❌ Not supported: *.google.com
```

## Chrome メモリセーバーとの共存

Chrome 108 以降、Chrome には **メモリセーバー**（`chrome://settings/performance`）という独自のタブ破棄機能が搭載されています。これはブラウザレベルで動作し、TabRest を含むいかなる拡張機能にも**問い合わせません**。

### これが意味すること

TabRest のホワイトリストは、**TabRest** に対してドメインをアンロードしないよう指示します。**Chrome** にタブを読み込んだまま維持するよう指示するものでは**ありません**。Chrome メモリセーバーが有効で、タブが十分に長く非アクティブだと判断した場合、そのドメインが TabRest のホワイトリストに載っていてもタブを破棄します。

これは Chrome プラットフォームの制約です。タブをメモリセーバーの対象外にする拡張機能 API は存在しません。

### この状況の兆候

- ドメインが TabRest のホワイトリストに載っている
- それでもタブが自動的にアンロードされる
- TabRest のサスペンド警告トーストが表示されずに破棄された（TabRest が実行していないため）

### タブを完全に保護する方法

次のうち**いずれか 1 つ**を選びます:

**オプション A（推奨）: Chrome メモリセーバーをオフにする**

1. `chrome://settings/performance` を開きます
2. **メモリセーバー**をオフに切り替えます
3. これで TabRest が完全に制御し、ホワイトリストが期待どおりに動作します

**オプション B: Chrome 内に 2 つ目のリストを維持する**

1. `chrome://settings/performance` を開きます
2. **常にこれらのサイトをアクティブにする**の下に、TabRest のホワイトリストにあるものと同じドメインを追加します
3. 両方のシステムがそれらのタブをスキップします

**自然に保護されるタブ**（設定なしで両方のシステムで機能します）:

- 音声または動画を再生しているタブ
- カメラまたはマイクを使用しているタブ
- 未保存のフォーム入力があるタブ
- 固定タブ

## プライバシー

ホワイトリストのエントリは:
- Chrome の同期ストレージにローカルで保存されます
- Chrome ブラウザ間で同期されます（同期が有効な場合）
- 外部サーバーに送信されることはありません
- 第三者と共有されることはありません

詳細は[プライバシーポリシー](/docs/privacy-policy)を参照してください。
