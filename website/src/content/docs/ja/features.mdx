# 機能

TabRest は、タブを管理してメモリを解放するための複数の方法を提供します。

**ポップアップ UI のヒント**: ポップアップ内の折りたたみ可能なセクション（タブ、クイック設定、その他のアクション、セッション、統計）は、ポップアップを閉じても開閉状態を記憶するため、繰り返しの操作で時間を節約できます。

## 自動アンロード

### タイマーベースのアンロード

一定時間の非アクティブ状態が続いたタブを自動的にアンロードします:

| タイマーオプション | ユースケース |
|--------------|----------|
| 5 分     | 最も積極的なメモリ節約 |
| 10 分    | 積極的なメモリ節約 |
| 15 分    | バランス重視の積極モード |
| 30 分    | デフォルトのバランスモード |
| 1 時間    | ライトユーザー向け |
| 2 時間    | タブを長めに保持 |
| 4 時間    | 最小限の自動アンロード |

**仕組み**: TabRest は各タブを最後に操作した時刻（クリック、スクロール、フォーカス）を追跡します。設定した時間が経過すると、非アクティブなタブがアンロードされます。

### メモリしきい値によるアンロード

システムの RAM 使用率がしきい値を超えたときにタブをアンロードします:

- **範囲**: メモリ使用率 60〜95%
- **デフォルト**: 80%
- **チェック間隔**: 30 秒ごと

トリガーされると、TabRest はメモリがしきい値を下回るまで、最も長く使われていない（LRU）タブをアンロードします。

### 起動時アンロード

Chrome の起動時にすぐメモリを解放します:

- **デフォルト**: 有効
- **動作**: ブラウザ起動時に、すべてのウィンドウの非アクティブなタブをすべて強制的にアンロードし、すべての保護（固定、ホワイトリスト、音声、フォーム）をバイパスします
- **役立つ場面**: 前回のセッションから多数のタブを復元する際に、メモリを無駄にしないようにします

### タブ数のしきい値

十分な数の非アクティブなタブがある場合にのみ自動アンロードを開始します:

- **範囲**: 3〜20 タブ
- **デフォルト**: 6 タブ
- **無効化**: 0 に設定すると常に自動アンロードします

**ユースケース**: タブが数個しか開いていないときの積極的なアンロードを防ぎます。

### アイドル状態時のみ

コンピュータがアイドル状態のときにのみタブを自動アンロードします:

- **デフォルト**: 無効
- **しきい値**: 2〜15 分の非アクティブ状態
- **検出**: Chrome のアイドル API を使用します（マウス/キーボードの操作を検出）

**ユースケース**: アクティブなブラウジングセッションを決して中断しません。

### タブごとのメモリしきい値

過剰なメモリを消費する個々のタブを自動的にアンロードします:

- **範囲**: 100MB〜1GB（JavaScript ヒープ）
- **デフォルト**: 無効
- **注意**: JS メモリのみを計測し、タブ全体のメモリは計測しません

**ユースケース**: 経過時間ではなく、メモリを多く消費するタブを狙って対象にします。

#### サスペンド警告トースト

自動破棄が実行される前に、ページ上に 3 秒間の警告を表示します:

- **デフォルト**: 有効
- **遅延**: 1〜10 秒で設定可能（デフォルト 3 秒）
- **動作**: 自動サスペンドの前に通知バーを表示します。次の操作でキャンセルできます:
  - タブに切り替える
  - ページで音声を再生する
  - フォームフィールドを編集する
  - タブをスヌーズする
- **理由**: 未保存の作業の消失や重要なタスクの中断を防ぎます

**例**: 未保存のテキストがあるタブから離れたとします。TabRest はアンロードの 3 秒前に警告を表示し、保存またはアンロードのキャンセルの時間を与えます。

## オフライン時はスキップ

ネットワークが利用できないときにタブが破棄されるのを防ぎます:

- **デフォルト**: 有効
- **動作**: `navigator.onLine` が false のとき自動アンロードを一時停止します
- **理由**: 破棄されたタブはネットワークなしでは再読み込みできないため、不便を避けられます

## グローバル一時停止

1 つのスイッチで、すべての自動タブアンロードを一時的に無効にします:

- **アクセス**: ポップアップとサイドパネルの上部付近にある一時停止バー
- **期間**: 30 分、1 時間、2 時間、または「再開するまで」（無期限）
- **一時停止中**: バーには**再開**ボタンと残り時間（例:「45 分後に再開」）が表示されます。無期限オプションの場合は「再開するまで」と表示されます
- **停止される対象**: 非アクティブタイマー、メモリしきい値、タブごとのメモリしきい値、起動時の自動アンロード
- **停止されない対象**: 手動アンロードアクション（現在/他/左/右）、右クリックのコンテキストメニュー、キーボードショートカット、ツールバークリックアクション。これらは常に動作します
- **インジケーター**: 一時停止中はツールバーバッジに ❚❚ が表示され、ポップアップのタブリストではタブごとのカウントダウンタイマーが非表示になります
- **永続性**: デバイスごとにローカルに保存されます（デバイス間で同期されません）。時間指定の一時停止は期限が過ぎると自動的に再開し、サービスワーカーの再起動後も維持されます

**スヌーズとの違い**: スヌーズは 1 つのタブまたはドメインを保護します。グローバル一時停止は、すべての自動アンロードを一度に停止する 1 つのスイッチです。

**ユースケース**: プレゼンテーション、複数タブでの多数のファイルのダウンロード、ゲーム、デバッグなど、しばらくタブをアンロードさせたくないあらゆる場面。

## 手動コントロール

ポップアップからタブのアンロードをオンデマンドで制御します:

| アクション | 説明 |
|--------|-------------|
| **現在のタブをアンロード** | アクティブなタブをアンロードします |
| **他のタブをアンロード** | 現在のタブ以外をすべてアンロードします |
| **左をアンロード** | 現在のタブより左のすべてのタブをアンロードします |
| **右をアンロード** | 現在のタブより右のすべてのタブをアンロードします |

ツールバーの TabRest アイコンまたは[キーボードショートカット](/docs/keyboard-shortcuts)からアクセスできます。

### 重複タブを閉じる

現在のウィンドウ内の重複したタブを自動的に閉じます:

- **アクセス**: ポップアップを開く → その他のアクション → 重複を閉じる
- **動作**: すべてのタブを URL でスキャンし、最も古いタブを残して新しい重複を閉じます
- **ユースケース**: 誤って開いた同じサイトの複数タブを素早く整理します

**例**: `github.com/user/project` を開いたタブが 3 つある場合、TabRest は新しい 2 つを閉じ、最初の 1 つを残します。

### タブ検索

ポップアップ内のタブに対するライブ検索フィルタです:

- **アクセス**: ポップアップを開く → 検索入力（表示に切り替え）
- **動作**: タブのタイトルと URL によるリアルタイムフィルタリング（大文字小文字を区別しません）
- **組み合わせ可能**: フィルタチップ（すべて/スリープ中/スヌーズ中/保護中）と併用できます
- **ユースケース**: 多くのタブを開いているときに特定のタブを素早く見つけます

**例**: 「github」と入力すると、GitHub 関連のすべてのタブがフィルタされます。

### ツールバークリックアクション

拡張機能アイコンをクリックしたときの動作をカスタマイズします:

| オプション | 動作 |
|--------|----------|
| **ポップアップを開く** | デフォルト。タブリストとクイック設定を表示します |
| **サイドパネルで開く** | ブラウザの常設サイドバーで TabRest を開きます（オプトイン） |
| **現在のタブを破棄** | 現在のタブをすぐにアンロードします |
| **他のタブを破棄** | 他のすべてのタブをすぐにアンロードします |

**ユースケース**: ポップアップを開かずにワンクリックの破棄や常設サイドバーへのアクセスを求めるパワーユーザー向けです。

## パワーモード

状況に応じて自動アンロードの積極度を調整します:

| モード | タイマー倍率 | メモリオフセット |
|------|------------------|---------------|
| **バッテリーセーバー** | 0.5 倍（速い） | -10%（しきい値を下げる） |
| **標準** | 1.0 倍（デフォルト） | 0% |
| **パフォーマンス** | 2.0 倍（遅い） | +10%（しきい値を上げる） |

**例**: タイマーが 30 分に設定されている場合:
- バッテリーセーバー: 15 分
- 標準: 30 分
- パフォーマンス: 60 分

## タブの保護

特定のタブを自動アンロードから保護します:

### 固定タブ
- **デフォルト**: 自動アンロードから保護されます
- **設定可能**: 設定で「固定タブをアンロード」を有効にすると上書きできます

### 音声/動画タブ
- **デフォルト**: メディア再生中は保護されます
- **検出**: `tab.audible` プロパティを確認します
- **設定可能**: 「音声/動画を再生しているタブを保護」を無効にするとアンロードを許可します

### フォームタブ
- **デフォルト**: フォームに未保存のデータがあるときは保護されます
- **検出**: コンテンツスクリプトが変更された入力フィールドとテキストエリアを確認します
- **権限**: オンデマンドのホスト権限（フォーム保護を有効にしたときのみリクエストされます）
- **設定可能**: 設定で「未保存フォームのあるタブを保護」を無効にするとアンロードを許可します
- **注意**: 標準的な HTML フォームで動作します。リッチテキストエディタ（TinyMCE、CKEditor）は検出できない場合があります。それらのサイトにはホワイトリストを使用してください

### ホワイトリスト
- **デフォルト**: YouTube、Gmail、Google ドキュメント、Miro、Figma、Notion
- **カスタム**: 任意のサイトを追加して自動アンロードを防ぎます
- **詳細**: [ホワイトリスト設定](/docs/whitelist-config)を参照してください

### タブ/ドメインのスヌーズ

特定のタブまたはドメイン全体を自動破棄から一時的に保護します:

| スヌーズオプション | 期間 |
|---------------|----------|
| タブスヌーズ | 30 分、1 時間、2 時間 |
| ドメインスヌーズ | 1 時間 |

**アクセス**: ページを右クリック → スヌーズオプション、またはポップアップのタブリストで ⏸️ ボタンを使用します。

**ユースケース**: あるサイトを恒久的なホワイトリストに追加せずに集中して作業するとき。

## サイドパネルモード

ポップアップではなく、Chrome の常設サイドバーで TabRest を開きます:

- **アクセス**: 設定 → ツールバーアクション → 「サイドパネルで開く」を切り替え
- **デフォルト**: 無効（オプトイン機能）
- **動作**: タブを切り替えても表示されたままのサイドバーパネルを開きます
- **レスポンシブ**: パネルは異なる画面サイズに合わせてレイアウトを調整します
- **元に戻す**: オフに切り替えるとポップアップモードに戻ります
- **要件**: サイドパネル API に対応する Chrome 114 以降

**ユースケース**: 操作のたびにパネルを閉じることなく、タブコントロールや統計に素早くアクセスできるよう TabRest を常に表示しておきたい場合。

## タブグループ

タブグループ全体を一度にアンロードします:

1. Chrome で**タブグループを作成**します（タブを右クリック → 新しいグループに追加）
2. **TabRest ポップアップを開き**、グループ名をクリックします
3. **グループ内のすべてのタブ**が一緒にアンロードされます

**注意**: タブグループは Chrome の機能です（Chrome 88 以降）。TabRest は既存のグループと連携します。

## インポートとエクスポート

設定をバックアップおよび復元します:

- **エクスポートされる内容**: ホワイトリスト、ブラックリスト、保存されたセッション
- **形式**: JSON（クリップボードベースで、保存/共有が簡単）
- **スキーマバージョン**: v1（将来の移行に備えた設計）
- **インポートの動作**: 追加型（新しい項目が既存のリストに追加され、重複は自動的に削除されます）
- **アクセス**: 設定またはポップアップ → セッションセクション → エクスポート/インポートボタン
- **ユースケース**: ブラウザ間での設定の移行、大きな変更前のバックアップ、チームメンバーとのホワイトリストの共有

**例**: ホワイトリストを JSON としてエクスポートし、メールで送信して、別のデバイスでインポートします。

## 統計

時間の経過に伴うメモリの節約量を追跡します:

- **アンロードされたタブ**: インストール以降の合計数
- **節約されたメモリ**: 解放された推定 MB（タブのメモリ使用量から計算）
- **統計を表示**: 設定ページを開きます
- **統計をリセット**: 設定内のリセットボタンを使用します

**プライバシー**: すべての統計はローカルに保存されます。データがブラウザの外に出ることはありません。

## バッジインジケーター

拡張機能アイコンにアンロードされたタブの数を表示します:

- **デフォルト**: 有効
- **表示**: ツールバーアイコン上の小さな数字バッジ
- **無効化**: 設定で「バッジカウントを表示」をオフにします

## RAM 使用状況の表示

ポップアップのヘッダーで現在のシステム RAM 使用状況を表示します:

- **表示**: タブ統計とともに RAM 使用率を表示します
- **警告**: RAM が設定したしきい値を超えたときに強調表示します
- **メモリ推定**: RAM 統計にカーソルを合わせると、破棄されたタブごとに解放されたメモリの推定値（タブあたり平均約 150 MB）が表示されます
- **ツールチップ**: しきい値との比較と推定メモリ回復量を表示します

## スクロール位置の復元

タブが再読み込みされる際に、スクロール位置を自動的に保存および復元します:

- **デフォルト**: 有効
- **動作**: 破棄前に位置を保存し、再読み込み時に復元します
- **ストレージ**: ローカルストレージ。古いエントリを自動的にクリーンアップします（最大 100 件）

**ユースケース**: 長い記事を、読んでいたまさにその場所から読み続けられます。

## 自動アンロード通知

タブが自動的にアンロードされたときに通知を受け取ります:

- **デフォルト**: 無効
- **内容**: タブのタイトル、理由（タイマー/RAM）、詳細を表示します
- **サイレント**: 通知は音を鳴らしません

**ユースケース**: TabRest がバックグラウンドで何をしているかを把握し続けられます。

## リンクをサスペンド状態で開く

リンクをサスペンド（破棄）状態で直接開きます:

- **アクセス**: 任意のリンクを右クリック → 「リンクをサスペンドされたタブで開く」
- **動作**: タブを作成し、読み込みを待ってから、すぐに破棄します
- **ユースケース**: メモリを消費せずにリンクを後で読むために保存します

## 多言語サポート

TabRest は以下に対応しています:

- 🇺🇸 英語
- 🇻🇳 ベトナム語

言語は Chrome の設定から自動検出されます。

## 技術的な機能

### Chrome Manifest V3
- モダンなサービスワーカーアーキテクチャ
- 効率的なリソース使用
- 最新の Chrome 標準に準拠

### オプションのホスト権限
- フォーム保護やその他のホスト依存機能を有効にしたときのみ権限をリクエストします
- 他の機能を失うことなく、Chrome の設定でいつでも権限を取り消せます
- 機能有効化時の透明性のある権限プロンプト

### オンデマンドのフォーム検出
- フォームチェッカースクリプトはフォーム保護が有効なときのみ注入されます
- 無効なときはパフォーマンスへの影響を最小限に抑えます
- 変更された入力フィールド、テキストエリア、選択ドロップダウンを検出します

### ES モジュール
- ネイティブな JavaScript モジュール
- ビルドステップが不要
- 高速な読み込みと実行

### Chrome Alarms API
- サービスワーカーの終了後も維持されます
- 信頼性の高い定期チェック
- バッテリー効率の良いスケジューリング
